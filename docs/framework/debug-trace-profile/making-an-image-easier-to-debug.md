<span data-ttu-id="eaf39-103">Yönetilmeyen kod derlerken ayar IDE anahtarları veya komut satırı seçenekleri tarafından hata ayıklama için yürütülebilir görüntü yapılandırabilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="eaf39-103">When compiling unmanaged code, you can configure an executable image for debugging by setting IDE switches or command-line options.</span></span> <span data-ttu-id="eaf39-104">Örneğin, kullanabileceğiniz /**Zi** Visual c++ komut satırı seçeneği, hata ayıklama simge dosyaları (dosya uzantısı .pdb) yaymak üzere isteyebilir.</span><span class="sxs-lookup"><span data-stu-id="eaf39-104">For example, you can use the /**Zi** command-line option in Visual C++ to ask it to emit debug symbol files (file extension .pdb).</span></span> <span data-ttu-id="eaf39-105">Benzer şekilde, /**Od** iyileştirme devre dışı bırakmak için derleyici komut satırı seçeneği söyler.</span><span class="sxs-lookup"><span data-stu-id="eaf39-105">Similarly, the /**Od** command-line option tells the compiler to disable optimization.</span></span> <span data-ttu-id="eaf39-106">Ortaya çıkan kodu daha yavaş çalışır, ancak bu gerekli olmalıdır hata ayıklamak daha kolay olur.</span><span class="sxs-lookup"><span data-stu-id="eaf39-106">The resulting code runs more slowly, but is easier to debug, should this be necessary.</span></span>

 <span data-ttu-id="eaf39-111">Bu JIT yapılandırma iki boyutu vardır:</span><span class="sxs-lookup"><span data-stu-id="eaf39-111">This JIT configuration has two aspects:</span></span>  
  
-   <span data-ttu-id="eaf39-112">İzleme bilgileri oluşturmak için JIT Derleyici isteyebilir.</span><span class="sxs-lookup"><span data-stu-id="eaf39-112">You can request the JIT-compiler to generate tracking information.</span></span> <span data-ttu-id="eaf39-113">Hata ayıklayıcı için MSIL zincirinde makine kodu karşılığı ile eşleşen ve yerel değişkenleri ve işlev bağımsız değişkenleri depolandığı izlemek için mümkün kılar.</span><span class="sxs-lookup"><span data-stu-id="eaf39-113">This makes it possible for the debugger to match up a chain of MSIL with its machine code counterpart, and to track where local variables and function arguments are stored.</span></span>  <span data-ttu-id="eaf39-114">.NET Framework sürüm 2. 0'da, bu yüzden istekte gerek JIT Derleyici izleme bilgilerini her zaman oluşturur.</span><span class="sxs-lookup"><span data-stu-id="eaf39-114">In the .NET Framework version 2.0, the JIT compiler will always generate tracking information, so there is no need to request it.</span></span>  
  
-   <span data-ttu-id="eaf39-115">Sonuçta elde edilen makine kodu iyileştirmeyecek şekilde JIT Derleyici isteyebilir.</span><span class="sxs-lookup"><span data-stu-id="eaf39-115">You can request the JIT-compiler to not optimize the resulting machine code.</span></span>  
  
 <span data-ttu-id="eaf39-116">Normalde, MSIL oluşturur derleyici bu JIT Derleyici Seçenekleri uygun şekilde, IDE anahtarları veya belirttiğiniz, örneğin, komut satırı seçenekleri göre ayarlar /**Od**.</span><span class="sxs-lookup"><span data-stu-id="eaf39-116">Normally, the compiler that generates the MSIL sets these JIT-compiler options appropriately, based upon the IDE switches or command-line options you specify, for example, /**Od**.</span></span>  
  
 <span data-ttu-id="eaf39-117">Bazı durumlarda, böylece hata ayıklamak makine ürettiği kodu kolaydır JIT Derleyici davranışını değiştirmek isteyebilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="eaf39-117">In some cases, you might want to change the behavior of the JIT compiler so that the machine code it generates is easier to debug.</span></span> <span data-ttu-id="eaf39-118">Örneğin, JIT izleme perakende derleme bilgilerini oluşturmak veya en iyi duruma getirme denetlemek isteyebilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="eaf39-118">For example, you might want to generate JIT tracking information for a retail build or control optimization.</span></span> <span data-ttu-id="eaf39-119">Bir başlatma (.ini) dosyası bunu yapabilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="eaf39-119">You can do so with an initialization (.ini) file.</span></span>  
  
 <span data-ttu-id="eaf39-120">Örneğin, MyApp.exe hata ayıklamak istediğiniz derleme çağrılırsa, bu üç satırları içeriyorsa MyApp.exe ile aynı klasörde MyApp.ini adlı bir metin dosyası oluşturabilirsiniz:</span><span class="sxs-lookup"><span data-stu-id="eaf39-120">For example, if the assembly you want to debug is called MyApp.exe, then you can create a text file named MyApp.ini, in the same folder as MyApp.exe, which contains these three lines:</span></span>  
  
```

GenerateTrackingInfo=1  
AllowOptimize=0  
```  
  
 <span data-ttu-id="eaf39-121">Herhangi bir etmeksizin seçeneği 0 olarak varsayılan olarak ve her seçeneğin değeri, 0 veya 1'e ayarlayabilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="eaf39-121">You can set the value of each option to 0 or 1, and any absent option defaults to 0.</span></span> <span data-ttu-id="eaf39-122">Ayarı `GenerateTrackingInfo` 1 ve `AllowOptimize` kolay hata ayıklama 0 olarak sağlar.</span><span class="sxs-lookup"><span data-stu-id="eaf39-122">Setting `GenerateTrackingInfo` to 1 and `AllowOptimize` to 0 provides the easiest debugging.</span></span>  
  
> [!NOTE]
>  <span data-ttu-id="eaf39-123">.NET Framework sürüm 2. 0'da, JIT Derleyici her zaman izleme bilgilerini değerinden bağımsız oluşturur `GenerateTrackingInfo`; ancak, `AllowOptimize` değeri hala bir etkisi vardır.</span><span class="sxs-lookup"><span data-stu-id="eaf39-123">In the .NET Framework version 2.0, the JIT compiler always generates tracking information regardless of the value for `GenerateTrackingInfo`;
