# <a name="collection-type-dependency-properties"></a><span data-ttu-id="d2aa9-102">Koleksiyon Türü Bağımlılık Özellikleri</span><span class="sxs-lookup"><span data-stu-id="d2aa9-102">Collection-Type Dependency Properties</span></span>
<span data-ttu-id="d2aa9-103">Bu konu, kılavuz ve nasıl özellik türü bir koleksiyon türü olduğu bir bağımlılık özelliği uygulamak önerilen modeller sağlar.</span><span class="sxs-lookup"><span data-stu-id="d2aa9-103">This topic provides guidance and suggested patterns for how to implement a dependency property where the type of the property is a collection type.</span></span>  
  
 
  
<a name="implementing"></a>   
## <a name="implementing-a-collection-type-dependency-property"></a><span data-ttu-id="d2aa9-104">Bir koleksiyon türü bağımlılık özelliği uygulama</span><span class="sxs-lookup"><span data-stu-id="d2aa9-104">Implementing a Collection-Type Dependency Property</span></span>  
 <span data-ttu-id="d2aa9-105">Genel olarak, bir bağımlılık özelliği için belirlediğiniz takip ettiğiniz uygulama modeli olan bir [!INCLUDE[TLA2#tla_clr](../../../../includes/tla2sharptla-clr-md.md)] burada bu özellik tarafından yedeklendiği özelliğinin sarmalayıcı bir <xref:System.Windows.DependencyProperty> tanımlayıcısı yerine bir alan veya diğer yapı.</span><span class="sxs-lookup"><span data-stu-id="d2aa9-105">For a dependency property in general, the implementation pattern that you follow is that you define a [!INCLUDE[TLA2#tla_clr](../../../../includes/tla2sharptla-clr-md.md)] property wrapper, where that property is backed by a <xref:System.Windows.DependencyProperty> identifier rather than a field or other construct.</span></span> <span data-ttu-id="d2aa9-106">Bir koleksiyon türü özelliğini uygularken bu aynı düzeni uygular.</span><span class="sxs-lookup"><span data-stu-id="d2aa9-106">You follow this same pattern when you implement a collection-type property.</span></span> <span data-ttu-id="d2aa9-107">Ancak, koleksiyonu içinde yer alan türü kendisi olduğunda bir koleksiyon türü özellik desenle karmaşıklık getirir bir <xref:System.Windows.DependencyObject> veya <xref:System.Windows.Freezable> türetilmiş sınıf.</span><span class="sxs-lookup"><span data-stu-id="d2aa9-107">However, a collection-type property introduces some complexity to the pattern whenever the type that is contained within the collection is itself a <xref:System.Windows.DependencyObject> or <xref:System.Windows.Freezable> derived class.</span></span>

 <span data-ttu-id="d2aa9-109">Bağımlılık özelliği oluşturduğunuzda, özelliğin varsayılan değerini ilk alan değeri olarak belirtmeyin.</span><span class="sxs-lookup"><span data-stu-id="d2aa9-109">When you create a dependency property, you do not specify the property default value as the initial field value.</span></span> <span data-ttu-id="d2aa9-110">Bunun yerine, bağımlılık özelliği meta verileri aracılığıyla varsayılan değeri belirtin.</span><span class="sxs-lookup"><span data-stu-id="d2aa9-110">Instead, you specify the default value through the dependency property metadata.</span></span> <span data-ttu-id="d2aa9-111">Bağımlılık özelliği meta verilerinde belirtilen varsayılan değeri, özellik bir başvuru türü ise, örneği başına varsayılan bir değer değil; Bunun yerine bu türünün tüm örnekleri için geçerli bir varsayılan değerdir.</span><span class="sxs-lookup"><span data-stu-id="d2aa9-111">If your property is a reference type, the default value specified in dependency property metadata is not a default value per instance; ## <a name="reporting-binding-value-changes-from-collection-properties"></a><span data-ttu-id="d2aa9-129">Derleme özelliklerinden bağlama değer değişikliklerini raporlama</span><span class="sxs-lookup"><span data-stu-id="d2aa9-129">Reporting Binding Value Changes from Collection Properties</span></span>  
 <span data-ttu-id="d2aa9-130">Kendisini bir bağımlılık özelliği olan bir koleksiyon özelliği alt özelliklerine değişiklikleri otomatik olarak bildirmiyor.</span><span class="sxs-lookup"><span data-stu-id="d2aa9-130">A collection property that is itself a dependency property does not automatically report changes to its subproperties.</span></span> <span data-ttu-id="d2aa9-131">Derlemeye bağlamalar oluşturuyorsanız, bu nedenle bazı veri bağlama senaryoları geçersiz kılmalarını değişiklikleri, raporlama bağlama engelleyebilir.</span><span class="sxs-lookup"><span data-stu-id="d2aa9-131">If you are creating bindings into a collection, this can prevent the binding from reporting changes, thus invalidating some data binding scenarios.</span></span> <span data-ttu-id="d2aa9-132">Ancak, koleksiyon türü kullanıyorsanız <xref:System.Windows.FreezableCollection%601> , koleksiyon türü olarak sonra koleksiyondaki içerilen öğelerin alt değişiklikler düzgün bildirilir ve bağlama beklendiği gibi çalışır.</span><span class="sxs-lookup"><span data-stu-id="d2aa9-132">However, if you use the collection type <xref:System.Windows.FreezableCollection%601> as your collection type, then subproperty changes to contained elements in the collection are properly reported, and binding works as expected.</span></span>  
  
 <span data-ttu-id="d2aa9-133">Bağımlılık nesne koleksiyonundaki alt bağlama etkinleştirmek için koleksiyon özelliği türü olarak oluşturun <xref:System.Windows.FreezableCollection%601>, o koleksiyona herhangi bir tür kısıtlamasıyla <xref:System.Windows.DependencyObject> türetilmiş sınıf.</span><span class="sxs-lookup"><span data-stu-id="d2aa9-133">To enable subproperty binding in a dependency object collection, create the collection property as type <xref:System.Windows.FreezableCollection%601>, with a type constraint for that collection to any <xref:System.Windows.DependencyObject> derived class.</span></span>
