## <a name="word-choice"></a><span data-ttu-id="60029-104">Sözcük seçimi</span><span class="sxs-lookup"><span data-stu-id="60029-104">Word Choice</span></span>  
 <span data-ttu-id="60029-105">**✓ YAPMAK** kolay okunabilir tanımlayıcı adlarını seçin.</span><span class="sxs-lookup"><span data-stu-id="60029-105">**✓ DO** choose easily readable identifier names.</span></span>  
  
 <span data-ttu-id="60029-106">Örneğin, adında bir özellik `HorizontalAlignment` daha İngilizce-okunamaz daha `AlignmentHorizontal`.</span><span class="sxs-lookup"><span data-stu-id="60029-106">For example, a property named `HorizontalAlignment` is more English-readable than `AlignmentHorizontal`.</span></span>  
  
 <span data-ttu-id="60029-107">**✓ YAPMAK** okunabilirlik kısaltma ayrıcalık tanıma.</span><span class="sxs-lookup"><span data-stu-id="60029-107">**✓ DO** favor readability over brevity.</span></span>  
  
 <span data-ttu-id="60029-108">Özellik adı `CanScrollHorizontally` daha iyidir `ScrollableX` (x ekseni belirsiz bir referansı).</span><span class="sxs-lookup"><span data-stu-id="60029-108">The property name `CanScrollHorizontally` is better than `ScrollableX` (an obscure reference to the X-axis).</span></span>  
  
 <span data-ttu-id="60029-109">**X yok** alt çizgi, kısa çizgi veya herhangi bir alfasayısal olmayan karakterler kullanın.</span><span class="sxs-lookup"><span data-stu-id="60029-109">**X DO NOT** use underscores, hyphens, or any other nonalphanumeric characters.</span></span>  
  
 <span data-ttu-id="60029-110">**X yok** Macarca gösterimini kullanın.</span><span class="sxs-lookup"><span data-stu-id="60029-110">**X DO NOT** use Hungarian notation.</span></span>  
  
 <span data-ttu-id="60029-111">**KAÇININ x** kullanılan programlama dilleri anahtar sözcükleriyle yaygın olarak çakışma kimlikleri kullanarak.</span><span class="sxs-lookup"><span data-stu-id="60029-111">**X AVOID** using identifiers that conflict with keywords of widely used programming languages.</span></span>  
  
 <span data-ttu-id="60029-112">Ortak dil belirtimi (CLS), kural 4 göre tüm uyumlu diller söz konusu dilin anahtar sözcüğü bir tanımlayıcı olarak kullanmak adlandırılmış öğelere erişime izin veren bir mekanizma sağlamanız gerekir.</span><span class="sxs-lookup"><span data-stu-id="60029-112">According to Rule 4 of the Common Language Specification (CLS), all compliant languages must provide a mechanism that allows access to named items that use a keyword of that language as an identifier.</span></span> <span data-ttu-id="60029-113">Örneğin, C# ' ta, kullanır @ işareti bu durumda kaçış mekanizması olarak.</span><span class="sxs-lookup"><span data-stu-id="60029-113">C#, for example, uses the @ sign as an escape mechanism in this case.</span></span> <span data-ttu-id="60029-114">Ancak, ortak anahtar sözcükleri olmadan birden kaçış sırası içeren bir yöntem kullanmak çok daha zor olduğundan önlemek için iyi bir fikir hala olduğu.</span><span class="sxs-lookup"><span data-stu-id="60029-114">However, it is still a good idea to avoid common keywords because it is much more difficult to use a method with the escape sequence than one without it.</span></span>  
  
## <a name="using-abbreviations-and-acronyms"></a><span data-ttu-id="60029-115">Kısaltmalar ve kısaltmalar kullanma</span><span class="sxs-lookup"><span data-stu-id="60029-115">Using Abbreviations and Acronyms</span></span>  
 <span data-ttu-id="60029-116">**X yok** kısaltmalar veya kısaltmalar tanımlayıcı adları bir parçası olarak kullanın.</span><span class="sxs-lookup"><span data-stu-id="60029-116">**X DO NOT** use abbreviations or contractions as part of identifier names.</span></span>  
  
 <span data-ttu-id="60029-117">Örneğin, `GetWindow` yerine `GetWin`.</span><span class="sxs-lookup"><span data-stu-id="60029-117">For example, use `GetWindow` rather than `GetWin`.</span></span>  
  
 <span data-ttu-id="60029-118">**X yok** yaygın olarak kabul edilen ve, yalnızca gerekli olduğunda olmaları durumunda bile olmayan kısaltmalar kullanın.</span><span class="sxs-lookup"><span data-stu-id="60029-118">**X DO NOT** use any acronyms that are not widely accepted, and even if they are, only when necessary.</span></span>

 <span data-ttu-id="60029-121">Örneğin, `GetLength` daha iyi bir adı `GetInt`.</span><span class="sxs-lookup"><span data-stu-id="60029-121">For example, `GetLength` is a better name than `GetInt`.</span></span>  
  
 <span data-ttu-id="60029-122">**✓ YAPMAK** bir tanımlayıcı türü ötesinde anlamsal anlamı olduğunda nadir durumlarda dile özgü adı yerine bir genel CLR türü adı kullanın.</span><span class="sxs-lookup"><span data-stu-id="60029-122">**✓ DO** use a generic CLR type name, rather than a language-specific name, in the rare cases when an identifier has no semantic meaning beyond its type.</span></span>

 <span data-ttu-id="60029-187">**✓ YAPMAK** var olan bir API yeni sürümlerini oluştururken eski API için benzer bir ad kullanın.</span><span class="sxs-lookup"><span data-stu-id="60029-187">**✓ DO** use a name similar to the old API when creating new versions of an existing API.</span></span>

 <span data-ttu-id="60029-189">**✓ YAPMAK** var olan bir API yeni bir sürümünü belirtmek için bir önek yerine bir sonek ekleme tercih eder.</span><span class="sxs-lookup"><span data-stu-id="60029-189">**✓ DO** prefer adding a suffix rather than a prefix to indicate a new version of an existing API.</span></span>  
  
 <span data-ttu-id="60029-190">Bu belge, göz atarken bulma yardımcı olur veya IntelliSense kullanma.</span><span class="sxs-lookup"><span data-stu-id="60029-190">This will assist discovery when browsing documentation, or using Intellisense.</span></span> <span data-ttu-id="60029-191">Çoğu tarayıcılar ve IntelliSense tanımlayıcıları alfabetik olarak göstermek için API eski sürümü yeni API'leri yakın düzenleneceğini.</span><span class="sxs-lookup"><span data-stu-id="60029-191">The old version of the API will be organized close to the new APIs, because most browsers and Intellisense show identifiers in alphabetical order.</span></span>
