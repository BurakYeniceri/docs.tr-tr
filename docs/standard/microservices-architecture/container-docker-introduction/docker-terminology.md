# <a name="docker-terminology"></a><span data-ttu-id="58288-103">Docker terimleri</span><span class="sxs-lookup"><span data-stu-id="58288-103">Docker terminology</span></span>

<span data-ttu-id="58288-104">Bu bölümde, terimleri ve tanımları Docker derin almadan önce bilmeniz listelenir.</span><span class="sxs-lookup"><span data-stu-id="58288-104">This section lists terms and definitions you should be familiar with before getting deeper into Docker.</span></span> <span data-ttu-id="58288-105">Daha fazla tanımları için kapsamlı bakın [sözlüğü](https://docs.docker.com/glossary/) Docker tarafından sağlanan.</span><span class="sxs-lookup"><span data-stu-id="58288-105">For further definitions, see the extensive [glossary](https://docs.docker.com/glossary/) provided by Docker .</span></span>

<span data-ttu-id="58288-106">**Kapsayıcı görüntüsü**: tüm bağımlılıkları ve kapsayıcı oluşturmak için gereken bilgileri içeren bir paket.</span><span class="sxs-lookup"><span data-stu-id="58288-106">**Container image**: A package with all the dependencies and information needed to create a container.</span></span> <span data-ttu-id="58288-107">Görüntüyü bir kapsayıcı çalışma zamanı tarafından kullanılacak dağıtım ve yürütme yapılandırması artı (örneğin, çerçeveler) tüm bağımlılıkları içerir.</span><span class="sxs-lookup"><span data-stu-id="58288-107">An image includes all the dependencies (such as frameworks) plus deployment and execution configuration to be used by a container runtime.</span></span> <span data-ttu-id="58288-108">Genellikle, görüntüyü kapsayıcının dosya sistemi oluşturmak için birbirinin Yığılmış katmanlardır birden çok temel görüntüleri türetir.</span><span class="sxs-lookup"><span data-stu-id="58288-108">Usually, an image derives from multiple base images that are layers stacked on top of each other to form the container's filesystem.</span></span> <span data-ttu-id="58288-109">Görüntü oluşturulduktan sonra sabittir.</span><span class="sxs-lookup"><span data-stu-id="58288-109">An image is immutable once it has been created.</span></span>

<span data-ttu-id="58288-110">**Dockerfile**: bir Docker görüntüsü oluşturma hakkında yönergeler içeren bir metin dosyası.</span><span class="sxs-lookup"><span data-stu-id="58288-110">**Dockerfile**: A text file that contains instructions for how to build a Docker image.</span></span> <span data-ttu-id="58288-111">Bir toplu betik gibi olan ilk satırını başından itibaren temel görüntü durumları ve ardından gerekli programları yüklemek, vb. dosyaları kopyalamak için yönergeleri izleyin, çalışma ortamı elde edene kadar ihtiyacınız.</span><span class="sxs-lookup"><span data-stu-id="58288-111">It's like a batch script, the first line states the base image to begin with and then follow the instructions to install required programs, copy files and so on, until you get the working environment you need.</span></span>

<span data-ttu-id="58288-112">**Derleme**: bir kapsayıcı görüntüsü oluşturma eylemini temel bilgileri ve kendi Dockerfile, artı ek görüntünün nerede üretilmiştir klasördeki dosyaları tarafından sağlanan bağlam.</span><span class="sxs-lookup"><span data-stu-id="58288-112">**Build**: The action of building a container image based on the information and context provided by its Dockerfile, plus additional files in the folder where the image is built.</span></span> <span data-ttu-id="58288-113">Docker görüntüleri oluşturabileceğinizi **docker derleme** komutu.</span><span class="sxs-lookup"><span data-stu-id="58288-113">You can build images with the Docker **docker build** command.</span></span> 

<span data-ttu-id="58288-114">**Kapsayıcı**: bir Docker görüntüsü örneği.</span><span class="sxs-lookup"><span data-stu-id="58288-114">**Container**: An instance of a Docker image.</span></span> <span data-ttu-id="58288-115">Bir kapsayıcıyı tek bir uygulama, işlem veya hizmet yürütülmesini temsil eder.</span><span class="sxs-lookup"><span data-stu-id="58288-115">A container represents the execution of a single application, process, or service.</span></span> <span data-ttu-id="58288-116">Bu, bir Docker görüntüsü, bir yürütme ortamı ve standart bir yönerge içeriğini oluşur.</span><span class="sxs-lookup"><span data-stu-id="58288-116">It consists of the contents of a Docker image, an execution environment, and a standard set of instructions.</span></span> <span data-ttu-id="58288-117">Hizmet ölçeklendirme aynı görüntüsünden bir kapsayıcı birden çok örneğini oluşturun.</span><span class="sxs-lookup"><span data-stu-id="58288-117">When scaling a service, you create multiple instances of a container from the same image.</span></span> <span data-ttu-id="58288-118">Veya toplu iş için her örnek farklı parametre geçirme aynı görüntüden birden çok kapsayıcı oluşturabilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="58288-118">Or a batch job can create multiple containers from the same image, passing different parameters to each instance.</span></span>

<span data-ttu-id="58288-119">**Birimleri**: kapsayıcı kullanabileceğiniz bir yazılabilir dosya sistemi sağlar.</span><span class="sxs-lookup"><span data-stu-id="58288-119">**Volumes**: Offer a writable filesystem that the container can use.</span></span> <span data-ttu-id="58288-120">Program bir yazılabilir dosya sistemi erişimi için birimler görüntüleri salt okunurdur ancak çoğu program, dosya sisteminde yazmanıza gerek olmadığından, kapsayıcı görüntüsü üzerine yazılabilir bir katman ekleyin.</span><span class="sxs-lookup"><span data-stu-id="58288-120">Since images are read-only but most programs need to write to the filesystem, volumes add a writable layer, on top of the container image, so the programs have access to a writable filesystem.</span></span> <span data-ttu-id="58288-121">Program, katmanlı bir dosya sistemi erişim, yalnızca dosya sistemi zamanki olduğunu bilmez.</span><span class="sxs-lookup"><span data-stu-id="58288-121">The program doesn't know it is accessing a layered filesystem, it is just the filesystem as usual.</span></span> <span data-ttu-id="58288-122">Birimler, konak sisteminde canlı ve Docker tarafından yönetilir.</span><span class="sxs-lookup"><span data-stu-id="58288-122">Volumes live in the host system and are managed by Docker.</span></span>

<span data-ttu-id="58288-123">**Etiket**: işareti ya da etiket böylece farklı bir görüntü veya aynı görüntünün (bağlı olarak sürüm numarasını veya hedef ortamı) sürümü tanımlanabilir görüntüleri uygulayabilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="58288-123">**Tag**: A mark or label you can apply to images so that different images or versions of the same image (depending on the version number or the target environment) can be identified.</span></span>

<span data-ttu-id="58288-124">**Çok aşamalı derleme**: son görüntü boyutunu küçültmek için yardımcı olan bir özellik, Docker 17.05 veya üzeri.</span><span class="sxs-lookup"><span data-stu-id="58288-124">**Multi-stage Build**: Is a feature, since Docker 17.05 or higher, that helps to reduce the size of the final images.</span></span> <span data-ttu-id="58288-125">Birkaç cümle ile sorunu çok aşamalı derleme ile Örneğin, büyük bir temel görüntüsü derleme ve uygulamayı yayımlama ve ardından yayımlama klasörüne bir çok daha küçük son görüntü oluşturmak için bir küçük yalnızca çalışma zamanı temel görüntü ile kullanmak için SDK'sını içeren kullanabilirsiniz</span><span class="sxs-lookup"><span data-stu-id="58288-125">In a few sentences, with multi-stage build you can use, for example, a large base image, containing the SDK, for compiling and publishing the application and then using the publishing folder with a small runtime-only base image, to produce a much smaller final image</span></span>

<span data-ttu-id="58288-126">**Depo (depo)**: görüntü sürümünü belirtir bir etiketle etiketlenmiş ilgili Docker görüntülerini koleksiyonu.</span><span class="sxs-lookup"><span data-stu-id="58288-126">**Repository (repo)**: A collection of related Docker images, labeled with a tag that indicates the image version.</span></span> <span data-ttu-id="58288-127">Bazı depolar, bir SDK'ları (Ağır), yalnızca çalışma zamanları (açık) içeren bir görüntü içeren görüntüsü, vb. gibi belirli bir görüntünün birden fazla çeşitlemelerini içerir. Bu çeşitleri etiketlerle işaretlenebilir.</span><span class="sxs-lookup"><span data-stu-id="58288-127">Some repos contain multiple variants of a specific image, such as an image containing SDKs (heavier), an image containing only runtimes (lighter), etc. Those variants can be marked with tags.</span></span> <span data-ttu-id="58288-128">Tek bir depoda bir Linux görüntüsü ve bir Windows görüntüsünü gibi platformu çeşitleri içerebilir.</span><span class="sxs-lookup"><span data-stu-id="58288-128">A single repo can contain platform variants, such as a Linux image and a Windows image.</span></span>

<span data-ttu-id="58288-129">**Kayıt defteri**: depolarına erişimi sağlayan bir hizmet.</span><span class="sxs-lookup"><span data-stu-id="58288-129">**Registry**: A service that provides access to repositories.</span></span> <span data-ttu-id="58288-130">Varsayılan kayıt defteri en genel görüntülerde [Docker Hub](https://hub.docker.com/) (kuruluş olarak Docker tarafından sahip olunan).</span><span class="sxs-lookup"><span data-stu-id="58288-130">The default registry for most public images is [Docker Hub](https://hub.docker.com/) (owned by Docker as an organization).</span></span> <span data-ttu-id="58288-131">Bir kayıt defteri, genellikle birden çok takımı depolarından içerir.</span><span class="sxs-lookup"><span data-stu-id="58288-131">A registry usually contains repositories from multiple teams.</span></span> <span data-ttu-id="58288-132">Şirketler genellikle oluşturmuş olduğunuz görüntülerini depolayıp yönetin için özel kayıt defterleri sahiptir.</span><span class="sxs-lookup"><span data-stu-id="58288-132">Companies often have private registries to store and manage images they've created.</span></span> <span data-ttu-id="58288-133">Azure Container Registry, başka bir örnektir.</span><span class="sxs-lookup"><span data-stu-id="58288-133">Azure Container Registry is another example.</span></span>

<span data-ttu-id="58288-134">**Çok yay görüntü**: çok mimari için uygun görüntü seçimi basitleştiren bir özelliktir Docker çalıştırdığı platforma göre örneğin bir Dockerfile istediğinde bir temel görüntü **microsoft/dotnet gelen : 2.1-sdk** gerçekten alır kayıt defterinden **2.1-sdk-nanoserver-1709**, **2.1-sdk-nanoserver-1803** veya **2.1-sdk-alpine**bağlı olarak işletim sistemi ve Docker'ın çalıştığı sürümü.</span><span class="sxs-lookup"><span data-stu-id="58288-134">**Multi-arch image**: For multi-architecture, is a feature that simplifies the selection of the appropriate image, according to the platform where Docker is running, e.g. when a Dockerfile requests a base image **FROM microsoft/dotnet:2.1-sdk** from the registry it actually gets **2.1-sdk-nanoserver-1709**, **2.1-sdk-nanoserver-1803** or **2.1-sdk-alpine**, depending on the operating system and version where Docker is running.</span></span>

<span data-ttu-id="58288-135">**Docker Hub**: görüntüleri karşıya yüklemek ve bunlarla çalışmak için genel bir kayıt defteri.</span><span class="sxs-lookup"><span data-stu-id="58288-135">**Docker Hub**: A public registry to upload images and work with them.</span></span> <span data-ttu-id="58288-136">Docker Hub, Docker görüntüsünü barındıran, genel veya özel kayıt defterleri, yapı tetikleyicilerini ve web kancaları ve GitHub ve Bitbucket ile tümleştirme sağlar.</span><span class="sxs-lookup"><span data-stu-id="58288-136">Docker Hub provides Docker image hosting, public or private registries, build triggers and web hooks, and integration with GitHub and Bitbucket.</span></span>

<span data-ttu-id="58288-137">**Azure Container Registry**: Docker görüntülerini ve bileşenlerinin azure'da çalışmaya yönelik bir genel kaynağı.</span><span class="sxs-lookup"><span data-stu-id="58288-137">**Azure Container Registry**: A public resource for working with Docker images and its components in Azure.</span></span> <span data-ttu-id="58288-138">Bu mümkün hale erişimi denetlemenizi sağlar ve Azure dağıtımlarınızı yakın olan bir kayıt defteri sağlar, Azure Active Directory grupları ve izinleri kullanın.</span><span class="sxs-lookup"><span data-stu-id="58288-138">This provides a registry that is close to your deployments in Azure and that gives you control over access, making it possible to use your Azure Active Directory groups and permissions.</span></span>

<span data-ttu-id="58288-139">**Docker Trusted Registry (DTR)**: olabilecek bir Docker kayıt defteri hizmetinden (Docker), kuruluşunuzun veri merkezi ve ağ içinde bulunduğu için şirket içi yüklü.</span><span class="sxs-lookup"><span data-stu-id="58288-139">**Docker Trusted Registry (DTR)**: A Docker registry service (from Docker) that can be installed on-premises so it lives within the organization's datacenter and network.</span></span> <span data-ttu-id="58288-140">Kuruluş içinde yönetilen özel görüntüleri uygundur.</span><span class="sxs-lookup"><span data-stu-id="58288-140">It is convenient for private images that should be managed within the enterprise.</span></span> <span data-ttu-id="58288-141">Docker Trusted Registry, Docker Datacenter ürünün bir parçası olarak dahil edilir.</span><span class="sxs-lookup"><span data-stu-id="58288-141">Docker Trusted Registry is included as part of the Docker Datacenter product.</span></span> <span data-ttu-id="58288-142">Daha fazla bilgi için [Docker Trusted Registry (DTR)](https://docs.docker.com/docker-trusted-registry/overview/).</span><span class="sxs-lookup"><span data-stu-id="58288-142">For more information, see [Docker Trusted Registry (DTR)](https://docs.docker.com/docker-trusted-registry/overview/).</span></span>

<span data-ttu-id="58288-143">**Docker Community Edition'ı (CE)**: Windows ve macOS için oluşturma, çalıştırma ve kapsayıcıları yerel olarak test etme için geliştirme araçları.</span><span class="sxs-lookup"><span data-stu-id="58288-143">**Docker Community Edition (CE)**: Development tools for Windows and macOS for building, running, and testing containers locally.</span></span> <span data-ttu-id="58288-144">İçin docker CE Windows, Linux ve Windows kapsayıcıları için geliştirme ortamları sağlar.</span><span class="sxs-lookup"><span data-stu-id="58288-144">Docker CE for Windows provides development environments for both Linux and Windows Containers.</span></span> <span data-ttu-id="58288-145">Windows üzerinde Linux Docker konağı dayalı bir [Hyper-V](https://www.microsoft.com/cloud-platform/server-virtualization) sanal makine.</span><span class="sxs-lookup"><span data-stu-id="58288-145">The Linux Docker host on Windows is based on a [Hyper-V](https://www.microsoft.com/cloud-platform/server-virtualization) virtual machine.</span></span> <span data-ttu-id="58288-146">Windows kapsayıcıları için konak, doğrudan Windows üzerinde temel alır.</span><span class="sxs-lookup"><span data-stu-id="58288-146">The host for Windows Containers is directly based on Windows.</span></span> <span data-ttu-id="58288-147">Mac için docker CE Apple hiper yönetici Framework'te temel alır ve [xhyve hiper yönetici](https://github.com/mist64/xhyve), üzerinde Oracle temel Docker araç değiştirir Linux Docker konak sanal makine için Mac OS x Docker CE Windows ve Mac sağlar VirtualBox.</span><span class="sxs-lookup"><span data-stu-id="58288-147">Docker CE for Mac is based on the Apple Hypervisor framework and the [xhyve hypervisor](https://github.com/mist64/xhyve), which provides a Linux Docker host virtual machine on Mac OS X. Docker CE for Windows and for Mac replaces Docker Toolbox, which was based on Oracle VirtualBox.</span></span>

<span data-ttu-id="58288-148">**Docker Enterprise Edition (EE)**: bir geliştirme Linux ve Windows için Docker araçları kurumsal ölçekli sürümü.</span><span class="sxs-lookup"><span data-stu-id="58288-148">**Docker Enterprise Edition (EE)**: An enterprise-scale version of Docker tools for Linux and Windows development.</span></span>

<span data-ttu-id="58288-149">**Compose**: tanımlama ve çok kapsayıcılı uygulamalar çalıştırmak için meta veri biçimiyle bir komut satırı aracı ve YAML dosyası.</span><span class="sxs-lookup"><span data-stu-id="58288-149">**Compose**: A command-line tool and YAML file format with metadata for defining and running multi-container applications.</span></span> <span data-ttu-id="58288-150">Birden çok ortama bağlı olarak değerleri geçersiz kılabilir bir veya daha fazla .yml dosyaları görüntülerle dayalı tek bir uygulama tanımlarsınız.</span><span class="sxs-lookup"><span data-stu-id="58288-150">You define a single application based on multiple images with one or more .yml files that can override values depending on the environment.</span></span> <span data-ttu-id="58288-151">Tanımları oluşturduktan sonra tüm çok kapsayıcılı bir uygulama tek bir komutla dağıtabilirsiniz (docker-oluşturmanıza) Docker ana bilgisayarda görüntü her bir kapsayıcı oluşturur.</span><span class="sxs-lookup"><span data-stu-id="58288-151">After you have created the definitions, you can deploy the whole multi-container application with a single command (docker-compose up) that creates a container per image on the Docker host.</span></span>

<span data-ttu-id="58288-152">**Küme**: Docker ana bilgisayarları bir tek sanal Docker konağı değilmiş gibi uygulama hizmetleri birden fazla örneğe genişletebilir, kullanıma sunulan koleksiyonunu yayılan küme içindeki birden çok konak genelinde.</span><span class="sxs-lookup"><span data-stu-id="58288-152">**Cluster**: A collection of Docker hosts exposed as if it were a single virtual Docker host, so that the application can scale to multiple instances of the services spread across multiple hosts within the cluster.</span></span> <span data-ttu-id="58288-153">Kubernetes, Azure Service Fabric, Docker Swarm ve Mesosphere DC/OS ile docker kümeleri oluşturulabilir.</span><span class="sxs-lookup"><span data-stu-id="58288-153">Docker clusters can be created with Kubernetes, Azure Service Fabric, Docker Swarm and Mesosphere DC/OS.</span></span>
