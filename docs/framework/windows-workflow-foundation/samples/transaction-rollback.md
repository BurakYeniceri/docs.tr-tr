# <a name="transaction-rollback"></a><span data-ttu-id="d7a8e-102">İşlemi geri alma</span><span class="sxs-lookup"><span data-stu-id="d7a8e-102">Transaction Rollback</span></span>
<span data-ttu-id="d7a8e-103">Bu örnek, özel bir oluşturma işlemi gösterilmektedir <xref:System.Activities.NativeActivity> , ortam erişen <xref:System.Activities.RuntimeTransactionHandle> ortam işlem almak ve açıkça geri alma.</span><span class="sxs-lookup"><span data-stu-id="d7a8e-103">This sample shows how to create a custom <xref:System.Activities.NativeActivity> that accesses the ambient <xref:System.Activities.RuntimeTransactionHandle> to get the ambient transaction and explicitly roll it back.</span></span>  
  
## <a name="sample-details"></a><span data-ttu-id="d7a8e-104">Örnek Ayrıntıları</span><span class="sxs-lookup"><span data-stu-id="d7a8e-104">Sample Details</span></span>  
 <span data-ttu-id="d7a8e-105">İş akışında otomatik olarak bir işlem olduğu zaman tamamlandı en dıştaki <xref:System.Activities.Statements.TransactionScope> veya <xref:System.ServiceModel.Activities.TransactedReceiveScope> tamamlar.</span><span class="sxs-lookup"><span data-stu-id="d7a8e-105">In workflow, a transaction is automatically completed when the outermost <xref:System.Activities.Statements.TransactionScope> or <xref:System.ServiceModel.Activities.TransactedReceiveScope> completes.</span></span>  <span data-ttu-id="d7a8e-106">Geri kapsam sınırında işlenmeyen bir özel durum yayar, bir işlem örtülü olarak yapar.</span><span class="sxs-lookup"><span data-stu-id="d7a8e-106">A transaction implicitly rolls back when an unhandled exception propagates across the scope boundary.</span></span> <span data-ttu-id="d7a8e-107">Ancak, açıkça bir işlem özel durum gerek kalmadan geri almak için mantıklı zaman zamanlar olabilir.</span><span class="sxs-lookup"><span data-stu-id="d7a8e-107">However, there may be times when it makes sense to explicitly roll back a transaction without having to throw an exception.</span></span> <span data-ttu-id="d7a8e-108">Bu durumda, bu örnekteki gibi özel bir geri alma etkinlik açıkça ortam işlemi durdurmak ve isteğe bağlı bir özel durum nedeni sağlamak için kullanabilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="d7a8e-108">In this case, you can use the custom rollback activity like the one in this sample to explicitly abort the ambient transaction and provide an optional exception reason.</span></span>  
  
 <span data-ttu-id="d7a8e-109">`RollbackActivity` Olduğu bir <xref:System.Activities.NativeActivity> ortam almak için yürütme özelliklerine erişim gerektirdiğinden <xref:System.Activities.RuntimeTransactionHandle>.</span><span class="sxs-lookup"><span data-stu-id="d7a8e-109">The `RollbackActivity` is a <xref:System.Activities.NativeActivity> as it requires access to the execution properties to get the ambient <xref:System.Activities.RuntimeTransactionHandle>.</span></span> <span data-ttu-id="d7a8e-110">İçinde `Execute` yöntemi alır <xref:System.Activities.RuntimeTransactionHandle> ve olup olmadığını denetler `null`, etkinlik bir ortam çalıştırma işlem kullanıldığını gösterir.</span><span class="sxs-lookup"><span data-stu-id="d7a8e-110">In the `Execute` method, it obtains the <xref:System.Activities.RuntimeTransactionHandle> and checks whether it is `null`, which indicates that the activity was used without an ambient run-time transaction.</span></span> <span data-ttu-id="d7a8e-111">Daha sonra tekrar denetleme işlemin alır olmadığını `null` mevcuttur.</span><span class="sxs-lookup"><span data-stu-id="d7a8e-111">It then obtains the transaction, again checking whether `null` is present.</span></span> <span data-ttu-id="d7a8e-112">Bir ortam olma olasılığı vardır <xref:System.Activities.RuntimeTransactionHandle> olmadan hiç olmadığı kadar çalıştırma işlemi başlatılıyor.</span><span class="sxs-lookup"><span data-stu-id="d7a8e-112">It is possible to have an ambient <xref:System.Activities.RuntimeTransactionHandle> without ever initializing a run-time transaction.</span></span> <span data-ttu-id="d7a8e-113">Son olarak çağırarak işlem durdurur <xref:System.Transactions.Transaction.Rollback%2A> ve kullanıcı tarafından sağlanan bir özel durum ya da etkinliğin işlem geri alınacak bildiren genel bir özel durum belirterek.</span><span class="sxs-lookup"><span data-stu-id="d7a8e-113">Finally it aborts the transaction by calling <xref:System.Transactions.Transaction.Rollback%2A> and specifying either a user-provided exception or a generic exception that states that the activity rolled back the transaction.</span></span>  
  
 <span data-ttu-id="d7a8e-114">Tanıtım iş akışı oluşan bir <xref:System.Activities.Statements.TransactionScope> gövde önce ve sonra işlem durumu yazdırır `RollbackActivity` yürütür.</span><span class="sxs-lookup"><span data-stu-id="d7a8e-114">The demonstration workflow consists of a <xref:System.Activities.Statements.TransactionScope> whose body prints the transaction status before and after the `RollbackActivity` executes.</span></span> <span data-ttu-id="d7a8e-115">İşlem geri alındı olsa bile unutmayın <xref:System.Activities.Statements.TransactionScope> tamamlanana kadar yürütür ve gövde tamamlanana kadar iş akışını iptal değil.</span><span class="sxs-lookup"><span data-stu-id="d7a8e-115">Note that even though the transaction has been rolled back, the <xref:System.Activities.Statements.TransactionScope> executes to completion and does not abort the workflow until the body completes.</span></span> <span data-ttu-id="d7a8e-116">İş akışı olduğundan iptal <xref:System.Activities.Statements.TransactionScope.AbortInstanceOnTransactionFailure%2A> özelliği varsayılan olarak `true`.</span><span class="sxs-lookup"><span data-stu-id="d7a8e-116">The workflow is aborted because the <xref:System.Activities.Statements.TransactionScope.AbortInstanceOnTransactionFailure%2A> property defaults to `true`.</span></span>
