## <a name="class-enhancements"></a><span data-ttu-id="194b5-107">Sınıf geliştirmeleri</span><span class="sxs-lookup"><span data-stu-id="194b5-107">Class Enhancements</span></span>  
 <span data-ttu-id="194b5-108">Ana, bu geliştirmeler yinelenen ayırma kaçınma ve nesneleri eşitleme yüksek hacimli zaman uyumsuz yuva g/ç sırasında özelliğidir.</span><span class="sxs-lookup"><span data-stu-id="194b5-108">The main feature of these enhancements is the avoidance of the repeated allocation and synchronization of objects during high-volume asynchronous socket I/O.</span></span> <span data-ttu-id="194b5-109">Şu anda tarafından uygulanan başlangıç/bitiş tasarım deseni <xref:System.Net.Sockets.Socket> zaman uyumsuz yuva g/ç gerektirir sınıfı bir <xref:System.IAsyncResult?displayProperty=nameWithType> nesne için her zaman uyumsuz yuva işlemi ayrılan.</span><span class="sxs-lookup"><span data-stu-id="194b5-109">The Begin/End design pattern currently implemented by the <xref:System.Net.Sockets.Socket> class for asynchronous socket I/O requires a <xref:System.IAsyncResult?displayProperty=nameWithType> object be allocated for each asynchronous socket operation.</span></span>  
  
 <span data-ttu-id="194b5-110">Yeni <xref:System.Net.Sockets.Socket> sınıf geliştirmeleri, zaman uyumsuz yuva işlemleri tarafından yeniden kullanılabilir açıklanan <xref:System.Net.Sockets.SocketAsyncEventArgs?displayProperty=nameWithType> sınıf ayrılmış ve uygulama tarafından korunan nesneler.</span><span class="sxs-lookup"><span data-stu-id="194b5-110">In the new <xref:System.Net.Sockets.Socket> class enhancements, asynchronous socket operations are described by reusable <xref:System.Net.Sockets.SocketAsyncEventArgs?displayProperty=nameWithType> class objects allocated and maintained by the application.</span></span> <span data-ttu-id="194b5-111">Yüksek performanslı yuva uygulamaları en iyi Sürdürülen gerekir çakışan yuva işlemleri miktarını bilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="194b5-111">High-performance socket applications know best the amount of overlapped socket operations that must be sustained.</span></span> <span data-ttu-id="194b5-112">Uygulama, birçok oluşturabilirsiniz <xref:System.Net.Sockets.SocketAsyncEventArgs> gerekli nesneleri.</span><span class="sxs-lookup"><span data-stu-id="194b5-112">The application can create as many of the <xref:System.Net.Sockets.SocketAsyncEventArgs> objects that it needs.</span></span> <span data-ttu-id="194b5-113">Örneğin, bir sunucu uygulaması bekleyen işlemleri her zaman gelen istemci bağlantı hızları desteklemek için kabul 15 yuva olması gerekiyorsa, bunu yeniden kullanılabilir 15 ayırabilirsiniz <xref:System.Net.Sockets.SocketAsyncEventArgs> nesneleri önceden bu amaç için.</span><span class="sxs-lookup"><span data-stu-id="194b5-113">For example, if a server application needs to have 15 socket accept operations outstanding at all times to support incoming client connection rates, it can allocate 15 reusable <xref:System.Net.Sockets.SocketAsyncEventArgs> objects in advance for that purpose.</span></span>  
  
 <span data-ttu-id="194b5-114">Bu sınıf ile bir zaman uyumsuz yuva işlemi gerçekleştirmek için desen aşağıdaki adımlardan oluşur:</span><span class="sxs-lookup"><span data-stu-id="194b5-114">The pattern for performing an asynchronous socket operation with this class consists of the following steps:</span></span>

2.  <span data-ttu-id="194b5-116">Bağlam kümesi özellikleri olması (geri çağırma temsilci yöntemi ve veri arabelleği, örneğin) gerçekleştirilen hakkında çalışması için nesne.</span><span class="sxs-lookup"><span data-stu-id="194b5-116">Set properties on the context object to the operation about to be performed (the callback delegate method and data buffer, for example).</span></span>

4.  <span data-ttu-id="194b5-118">Zaman uyumsuz yuva yöntemi (xxxAsync) geri true değerini döndürür, tamamlanma durumu için içerik özelliklerine sorgu.</span><span class="sxs-lookup"><span data-stu-id="194b5-118">If the asynchronous socket method (xxxAsync) returns true in the callback, query the context properties for completion status.</span></span>  
  
5.  <span data-ttu-id="194b5-119">Zaman uyumsuz yuva yöntemi (xxxAsync) geri, işlem eşzamanlı olarak tamamlandı false değeri döndürülürse.</span><span class="sxs-lookup"><span data-stu-id="194b5-119">If the asynchronous socket method (xxxAsync) returns false in the callback, the operation completed synchronously.</span></span> <span data-ttu-id="194b5-120">Bağlam özellikleri için işlem sonucu sorgulanan.</span><span class="sxs-lookup"><span data-stu-id="194b5-120">The context properties may be queried for the operation result.</span></span>  
  
6.  <span data-ttu-id="194b5-121">Başka bir işlem bağlamı yeniden havuzu içinde beklemeye veya atmak.</span><span class="sxs-lookup"><span data-stu-id="194b5-121">Reuse the context for another operation, put it back in the pool, or discard it.</span></span>  
  
 <span data-ttu-id="194b5-122">Yeni zaman uyumsuz yuva işlemi bağlam nesnesinin ömrü uygulama kodundaki başvuruları ve zaman uyumsuz g/ç başvuruları tarafından belirlenir.</span><span class="sxs-lookup"><span data-stu-id="194b5-122">The lifetime of the new asynchronous socket operation context object is determined by references in the application code and asynchronous I/O references.</span></span> <span data-ttu-id="194b5-123">Uygulamanın zaman uyumsuz yuva işlemi yöntemlerden birini parametre olarak gönderildikten sonra bir zaman uyumsuz yuva işlemi bağlam nesnesi başvuru korumak gerekli değildir.</span><span class="sxs-lookup"><span data-stu-id="194b5-123">It is not necessary for the application to retain a reference to an asynchronous socket operation context object after it is submitted as a parameter to one of the asynchronous socket operation methods.</span></span> <span data-ttu-id="194b5-124">Tamamlama geri çağırma dönene kadar başvurulan kalır.</span><span class="sxs-lookup"><span data-stu-id="194b5-124">It will remain referenced until the completion callback returns.</span></span> <span data-ttu-id="194b5-125">Ancak gelecekteki zaman uyumsuz yuva işlemi için yeniden kullanılabilir içerik nesnesine başvuru tutulacak uygulama avantajlıdır.</span><span class="sxs-lookup"><span data-stu-id="194b5-125">However it is advantageous for the application to retain the reference to the context object so that it can be reused for a future asynchronous socket operation.</span></span>
