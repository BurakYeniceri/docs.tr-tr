<span data-ttu-id="12fb2-103">Öğesinden türetilen etkinlikleri <xref:System.Activities.Activity> veya alt sınıfları büyük iş akışlarıyla birleştirilerek oluşur veya doğrudan kod içinde oluşturulan.</span><span class="sxs-lookup"><span data-stu-id="12fb2-103">Activities that derive from <xref:System.Activities.Activity> or its subclasses can be composed into larger workflows, or created directly in code.</span></span> <span data-ttu-id="12fb2-104">Bu konu, kod veya Tasarımcısı'nda oluşturulan iş akışlarında özel etkinlikleri kullanmak açıklar.</span><span class="sxs-lookup"><span data-stu-id="12fb2-104">This topic describes how to use custom activities in workflows created either in code or in the designer.</span></span>  
  
> [!NOTE]
>  <span data-ttu-id="12fb2-105">Özel etkinlikler özel etkinlik ve kullandığı etkinlik (yani yapı işlemi tarafından oluşturulan bir instantiating türü tarafından yüklenen) derlendiğini sürece, bunlar tanımlanan projesinde kullanılabilir başvuru etkinlik yüklerse dinamik olarak (örneğin ActivityXAMLServices kullanarak), sonra farklı bir projedeki başvurulan derlemeyi yerleştirilmelidir veya tasarımcısı tarafından oluşturulan XAML el ile düzenleyebilirsiniz bunu etkinleştirmek için olması gerekir.</span><span class="sxs-lookup"><span data-stu-id="12fb2-105">Custom activities can be used in the same project in which they are defined, as long as both the custom activity and the activity that uses it are compiled (i.e. loaded by an instantiating type generated by the build process) If the referencing activity is loaded dynamically (e.g. using ActivityXAMLServices), then the referenced assembly should be placed in a different project, or the designer-generated XAML needs to be hand-edited to enable this.</span></span>
