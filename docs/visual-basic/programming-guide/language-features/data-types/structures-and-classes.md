-   <span data-ttu-id="3c6d4-116">Her ikisi de arabirimleri uygulayabilir.</span><span class="sxs-lookup"><span data-stu-id="3c6d4-116">Both can implement interfaces.</span></span>

## <a name="differences"></a><span data-ttu-id="3c6d4-120">Farkları</span><span class="sxs-lookup"><span data-stu-id="3c6d4-120">Differences</span></span>  
 <span data-ttu-id="3c6d4-121">Yapılar ve sınıflar aşağıdaki gerçekleştirilebilir farklılık gösterir:</span><span class="sxs-lookup"><span data-stu-id="3c6d4-121">Structures and classes differ in the following particulars:</span></span>  
  
-   <span data-ttu-id="3c6d4-122">Yapıları olan *değer türleri*; sınıflardır *başvuru türleri*.</span><span class="sxs-lookup"><span data-stu-id="3c6d4-122">Structures are *value types*; classes are *reference types*.</span></span> <span data-ttu-id="3c6d4-123">Yapı türünde bir değişken yapısı verileri içeren, yerine verileri bir sınıf türü olarak bir başvuru içeren.</span><span class="sxs-lookup"><span data-stu-id="3c6d4-123">A variable of a structure type contains the structure's data, rather than containing a reference to the data as a class type does.</span></span>  
  
-   <span data-ttu-id="3c6d4-124">Yığın ayırma yapıları kullanın; yığın ayırma sınıflarını kullanın.</span><span class="sxs-lookup"><span data-stu-id="3c6d4-124">Structures use stack allocation; classes use heap allocation.</span></span>  
  
-   <span data-ttu-id="3c6d4-125">Tüm yapısı öğeler `Public` varsayılan olarak; değişkenleri sınıf ve sabitler `Private` diğer sınıf üyeleri durumdayken varsayılan olarak, `Public` varsayılan olarak.</span><span class="sxs-lookup"><span data-stu-id="3c6d4-125">All structure elements are `Public` by default; class variables and constants are `Private` by default, while other class members are `Public` by default.</span></span> <span data-ttu-id="3c6d4-126">Sınıf üyeleri için bu davranış Visual Basic 6.0 sistem varsayılanlarını ile uyumluluk sağlar.</span><span class="sxs-lookup"><span data-stu-id="3c6d4-126">This behavior for class members provides compatibility with the Visual Basic 6.0 system of defaults.</span></span>  
  
-   <span data-ttu-id="3c6d4-127">Bir yapı en az bir paylaşılmayan değişkeni ya da paylaşılmayan, noncustom olmalıdır olay öğesi; bir sınıf tamamen boş olabilir.</span><span class="sxs-lookup"><span data-stu-id="3c6d4-127">A structure must have at least one nonshared variable or nonshared, noncustom event element; a class can be completely empty.</span></span>  
  
-   <span data-ttu-id="3c6d4-128">Yapı öğeleri olarak bildirilemez `Protected`; sınıf üyeleri olabilir.</span><span class="sxs-lookup"><span data-stu-id="3c6d4-128">Structure elements cannot be declared as `Protected`; class members can.</span></span>  
  
-   <span data-ttu-id="3c6d4-129">Yalnızca bir yapı yordamı olayları işleyebilirsiniz bir [paylaşılan](../../../../visual-basic/language-reference/modifiers/shared.md) `Sub` yordamı ve yalnızca, [AddHandler deyimi](../../../../visual-basic/language-reference/statements/addhandler-statement.md); herhangi bir sınıf yordamı ya da kullanarakolaylarıişleyebilir[ İşleme](../../../../visual-basic/language-reference/statements/handles-clause.md) anahtar sözcüğü veya `AddHandler` deyimi.</span><span class="sxs-lookup"><span data-stu-id="3c6d4-129">A structure procedure can handle events only if it is a [Shared](../../../../visual-basic/language-reference/modifiers/shared.md)`Sub` procedure, and only by means of the [AddHandler Statement](../../../../visual-basic/language-reference/statements/addhandler-statement.md); any class procedure can handle events, using either the [Handles](../../../../visual-basic/language-reference/statements/handles-clause.md) keyword or the `AddHandler` statement.</span></span> <span data-ttu-id="3c6d4-130">Daha fazla bilgi için bkz: [olayları](../../../../visual-basic/programming-guide/language-features/events/index.md).</span><span class="sxs-lookup"><span data-stu-id="3c6d4-130">For more information, see [Events](../../../../visual-basic/programming-guide/language-features/events/index.md).</span></span>  
  
-   <span data-ttu-id="3c6d4-131">Yapı değişken bildirimleri başlatıcıları ya da diziler için başlangıç boyutlar belirtemezsiniz; sınıf değişken bildirimleri olabilir.</span><span class="sxs-lookup"><span data-stu-id="3c6d4-131">Structure variable declarations cannot specify initializers or initial sizes for arrays; class variable declarations can.</span></span>

-   <span data-ttu-id="3c6d4-133">Yapıları devralınabilir değil; sınıflarıdır.</span><span class="sxs-lookup"><span data-stu-id="3c6d4-133">Structures are not inheritable; classes are.</span></span>  
  
-   <span data-ttu-id="3c6d4-134">Yapıları hiçbir zaman sonlandırıldı, ortak dil çalışma zamanı (CLR) hiçbir zaman çağırır <xref:System.Object.Finalize%2A> herhangi bir yapısını; yöntemini çağıran atıktoplayıcı tarafından (GC) sınıfları sona erdirildi <xref:System.Object.Finalize%2A> etkin başvuru vardır algıladığında bir sınıf Kalan.</span><span class="sxs-lookup"><span data-stu-id="3c6d4-134">Structures are never terminated, so the common language runtime (CLR) never calls the <xref:System.Object.Finalize%2A> method on any structure; classes are terminated by the garbage collector (GC), which calls <xref:System.Object.Finalize%2A> on a class when it detects there are no active references remaining.</span></span>  
  
-   <span data-ttu-id="3c6d4-135">Bir yapı bir oluşturucu gerektirmez; bir sınıf yapar.</span><span class="sxs-lookup"><span data-stu-id="3c6d4-135">A structure does not require a constructor; a class does.</span></span>  
  
-   <span data-ttu-id="3c6d4-136">Yapıları olabilir yalnızca parametreleri; izlerseniz paylaşılmayan oluşturucular sınıfları bunları ile veya parametresiz sahip olabilir.</span><span class="sxs-lookup"><span data-stu-id="3c6d4-136">Structures can have nonshared constructors only if they take parameters; classes can have them with or without parameters.</span></span>  
  
 <span data-ttu-id="3c6d4-137">Her yapısı daha genel bir parametresiz oluşturucuya örtük daha vardır.</span><span class="sxs-lookup"><span data-stu-id="3c6d4-137">Every structure has an implicit public constructor without parameters.</span></span> <span data-ttu-id="3c6d4-138">Bu oluşturucu, tüm yapısı veri öğeleri varsayılan değerlerine başlatır.</span><span class="sxs-lookup"><span data-stu-id="3c6d4-138">This constructor initializes all the structure's data elements to their default values.</span></span> <span data-ttu-id="3c6d4-139">Bu davranış tanımlanamaz.</span><span class="sxs-lookup"><span data-stu-id="3c6d4-139">You cannot redefine this behavior.</span></span>  
  
## <a name="instances-and-variables"></a><span data-ttu-id="3c6d4-140">Örnekler ve değişkenleri</span><span class="sxs-lookup"><span data-stu-id="3c6d4-140">Instances and Variables</span></span>  
 <span data-ttu-id="3c6d4-141">Yapılar değer türleri olduğundan, her bir yapı değişken kalıcı olarak tek tek yapısı örneğine bağlı.</span><span class="sxs-lookup"><span data-stu-id="3c6d4-141">Because structures are value types, each structure variable is permanently bound to an individual structure instance.</span></span> <span data-ttu-id="3c6d4-142">Ancak başvuru türleri sınıflardır ve bir nesne değişkeni çeşitli sınıf örnekleri farklı zamanlarda başvuruda bulunabilir.</span><span class="sxs-lookup"><span data-stu-id="3c6d4-142">But classes are reference types, and an object variable can refer to various class instances at different times.</span></span> <span data-ttu-id="3c6d4-143">Bu ayrım yapılar ve sınıflar kullanımınızı aşağıdaki şekillerde etkiler:</span><span class="sxs-lookup"><span data-stu-id="3c6d4-143">This distinction affects your usage of structures and classes in the following ways:</span></span>  
  
-   <span data-ttu-id="3c6d4-144">**başlatma.**</span><span class="sxs-lookup"><span data-stu-id="3c6d4-144">**Initialization.**</span></span> <span data-ttu-id="3c6d4-145">Yapısı değişkeni örtük olarak bir başlatma yapısı parametresiz bir kurucusu kullanarak tüm öğeleri içerir.</span><span class="sxs-lookup"><span data-stu-id="3c6d4-145">A structure variable implicitly includes an initialization of the elements using the structure's parameterless constructor.</span></span> <span data-ttu-id="3c6d4-146">Bu nedenle, `Dim s As struct1` eşdeğerdir `Dim s As struct1 = New struct1()`.</span><span class="sxs-lookup"><span data-stu-id="3c6d4-146">Therefore, `Dim s As struct1` is equivalent to `Dim s As struct1 = New struct1()`.</span></span>

-   <span data-ttu-id="3c6d4-150">**Hiçbir şey atama.**</span><span class="sxs-lookup"><span data-stu-id="3c6d4-150">**Assigning Nothing.**</span></span> <span data-ttu-id="3c6d4-151">Değer atayabilirsiniz [hiçbir şey](../../../../visual-basic/language-reference/nothing.md) değişkeni ancak örnek bir yapıya devam değişkeni ile ilişkili olmalıdır.</span><span class="sxs-lookup"><span data-stu-id="3c6d4-151">You can assign the value [Nothing](../../../../visual-basic/language-reference/nothing.md) to a structure variable, but the instance continues to be associated with the variable.</span></span> <span data-ttu-id="3c6d4-152">Hala yöntemlerini çağırın ve değişken öğeleri atama tarafından yeniden olsa da, kendi veri öğeleri erişebilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="3c6d4-152">You can still call its methods and access its data elements, although variable elements are reinitialized by the assignment.</span></span>

     <span data-ttu-id="3c6d4-157">Yapı öğeleri, ancak kendi örneğinde yalıtılır.</span><span class="sxs-lookup"><span data-stu-id="3c6d4-157">Structure elements, however, are isolated within their own instance.</span></span> <span data-ttu-id="3c6d4-158">Değişiklikleri değerlerine değil yansıtılır herhangi diğer yapı değişkenleri, aynı diğer durumlarda bile `Structure` bildirimi.</span><span class="sxs-lookup"><span data-stu-id="3c6d4-158">Changes to their values are not reflected in any other structure variables, even in other instances of the same `Structure` declaration.</span></span>  
  
-   <span data-ttu-id="3c6d4-159">**Eşitlik.**</span><span class="sxs-lookup"><span data-stu-id="3c6d4-159">**Equality.**</span></span> <span data-ttu-id="3c6d4-160">İki yapılarını eşitlik sınamasını ile öğesi öğeli test gerçekleştirilmesi gerekir.</span><span class="sxs-lookup"><span data-stu-id="3c6d4-160">Equality testing of two structures must be performed with an element-by-element test.</span></span> <span data-ttu-id="3c6d4-161">İki nesne değişkenleri kullanarak karşılaştırılabilir <xref:System.Object.Equals%2A> yöntemi.</span><span class="sxs-lookup"><span data-stu-id="3c6d4-161">Two object variables can be compared using the <xref:System.Object.Equals%2A> method.</span></span> <span data-ttu-id="3c6d4-162"><xref:System.Object.Equals%2A> iki değişken aynı örneğini olup olmadığını gösterir.</span><span class="sxs-lookup"><span data-stu-id="3c6d4-162"><xref:System.Object.Equals%2A> indicates whether the two variables point to the same instance.</span></span>
