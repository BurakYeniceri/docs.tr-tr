# <a name="compiler-error-cs1612"></a><span data-ttu-id="01eea-102">Derleyici Hatası CS1612</span><span class="sxs-lookup"><span data-stu-id="01eea-102">Compiler Error CS1612</span></span>
<span data-ttu-id="01eea-103">Bir değişken olmadığından 'ifadesindeki' dönüş değeri değiştirilemiyor</span><span class="sxs-lookup"><span data-stu-id="01eea-103">Cannot modify the return value of 'expression' because it is not a variable</span></span>  
  
 <span data-ttu-id="01eea-104">Bir ara ifade sonucu olarak üretilen ancak bir değişkende depolanmayan bir değer türü değiştirmek için girişimde bulunuldu.</span><span class="sxs-lookup"><span data-stu-id="01eea-104">An attempt was made to modify a value type that is produced as the result of an intermediate expression but is not stored in a variable.</span></span> <span data-ttu-id="01eea-105">Aşağıdaki örnekte gösterildiği gibi doğrudan bir genel koleksiyon yapıda değiştirme girişimi olduğunda bu hata oluşabilir:</span><span class="sxs-lookup"><span data-stu-id="01eea-105">This error can occur when you attempt to directly modify a struct in a generic collection, as shown in the following example:</span></span>  
  
```csharp  
List<myStruct> list = {…};  
list[0].Name = "MyStruct42"; //CS1612  
```  
  
 <span data-ttu-id="01eea-106">Yapı değiştirmek için ilk yerel bir değişkene atayın, değişkeni değiştirin, sonra değişken arka koleksiyondaki öğesine atayın.</span><span class="sxs-lookup"><span data-stu-id="01eea-106">To modify the struct, first assign it to a local variable, modify the variable, then assign the variable back to the item in the collection.</span></span>  
  
```csharp  
List<myStruct> list = {…};  
MyStruct ms = list[0];  
ms.Name = "MyStruct42";  
list[0] = ms;  
```  
  
 <span data-ttu-id="01eea-107">Değer türleri atamada kopyalanır bu hata oluşur.</span><span class="sxs-lookup"><span data-stu-id="01eea-107">This error occurs because value types are copied on assignment.</span></span> <span data-ttu-id="01eea-108">Bir özellik veya dizin oluşturucu için bir değer türü aldığınızda, bir başvuru değil nesnenin kendisini nesnesinin bir kopyasını aldıklarından.</span><span class="sxs-lookup"><span data-stu-id="01eea-108">When you retrieve a value type from a property or indexer, you are getting a copy of the object, not a reference to the object itself.</span></span> <span data-ttu-id="01eea-109">Gerçekte yöntemleri, depolama konumları (değişkenler) olduğundan döndürülen kopyalama özelliği veya dizin oluşturucu tarafından depolanmaz.</span><span class="sxs-lookup"><span data-stu-id="01eea-109">The copy that is returned is not stored by the property or indexer because they are actually methods, not storage locations (variables).</span></span> <span data-ttu-id="01eea-110">Bir değişken kopyaladığınız depolamalısınız değişiklik yapmadan önce bildirin.</span><span class="sxs-lookup"><span data-stu-id="01eea-110">You must store the copy into a variable that you declare before you can modify it.</span></span>  
  
 <span data-ttu-id="01eea-111">Bir özellik veya dizin oluşturucu bu durumda bir depolama konumunun varolan bir nesneye başvuru döndürdüğünden başvuru türleriyle hata oluşmaz.</span><span class="sxs-lookup"><span data-stu-id="01eea-111">The error does not occur with reference types because a property or indexer in that case returns a reference to an existing object, which is a storage location.</span></span>  
  
 <span data-ttu-id="01eea-112">Sınıf veya yapı tanımlıyorsanız, yapı üyelerine erişim sağlamak için özellik bildiriminizi değiştirerek bu hatayı çözebilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="01eea-112">If you are defining the class or struct, you can resolve this error by modifying your property declaration to provide access to the members of a struct.</span></span> <span data-ttu-id="01eea-113">İstemci kodu yazıyorsanız, kendi yapısı örneği oluşturarak, alanlarını değiştirerek ve sonra tüm yapısı özelliğine geri atayarak hatayı çözebilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="01eea-113">If you are writing client code, you can resolve the error by creating your own instance of the struct, modifying its fields, and then assigning the entire struct back to the property.</span></span> <span data-ttu-id="01eea-114">Üçüncü bir alternatif olarak, bir sınıf, yapı değiştirebilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="01eea-114">As a third alternative, you can change your struct to a class.</span></span>  
  
## <a name="example"></a><span data-ttu-id="01eea-115">Örnek</span><span class="sxs-lookup"><span data-stu-id="01eea-115">Example</span></span>  
 <span data-ttu-id="01eea-116">Aşağıdaki örnekte gösterildiği gibi yapının tamamını döndüren bir kapsayan sınıfı bir özellik aracılığıyla yapı üyesi erişmeyi denediğinde CS1612 da oluşur:</span><span class="sxs-lookup"><span data-stu-id="01eea-116">CS1612 also occurs when you attempt to access the member of a struct through a property on an enclosing class that is returning the entire struct, as shown in the following example:</span></span>  
  
```csharp  
// CS1612.cs  
using System;  
  
public struct MyStruct  
{  
    public int Width;  
}  
  
public class ListView  
{  
    MyStruct ms;  
    public MyStruct Size  
    {  
        get { return ms; }  
        set { ms = value; }  
    }  
}  
  
public class MyClass  
{  
    public MyClass()  
    {  
        ListView lvi;  
        lvi = new ListView();  
        lvi.Size.Width = 5; // CS1612  
  
        // You can use the following lines instead.  
        // MyStruct ms;  
        // ms.Width = 5;  
        // lvi.Size = ms;  // CS1612  
    }  
  
    public static void Main()   
    {  
        MyClass mc = new MyClass();  
        // Keep the console open in debug mode.  
        Console.WriteLine("Press any key to exit.");  
        Console.ReadKey();     
    }  
}  
```
