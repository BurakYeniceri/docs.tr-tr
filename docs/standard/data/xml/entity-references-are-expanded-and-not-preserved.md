<span data-ttu-id="6da32-103">Varlık başvurusu genişletilmiş ve temsil ettiği, metnin tarafından yerine **XmlEntityReference** düğümü oluşturulmadı.</span><span class="sxs-lookup"><span data-stu-id="6da32-103">When the entity reference is expanded and replaced by the text it represents, the **XmlEntityReference** node is not created.</span></span> <span data-ttu-id="6da32-104">Bunun yerine, varlık bildirimi ayrıştırılır ve bildiriminde içerikten oluşturulan düğümler yerine, kopyaladığınız **XmlEntityReference**.</span><span class="sxs-lookup"><span data-stu-id="6da32-104">Instead, the entity declaration is parsed, and nodes created from the content in the declaration are copied in the place of the **XmlEntityReference**.</span></span> <span data-ttu-id="6da32-105">Bu nedenle, içinde `&publisher;` örnek, `&publisher;` , ancak bunun yerine, kaydedilmez bir **XmlText** düğüm oluşturulur.</span><span class="sxs-lookup"><span data-stu-id="6da32-105">Therefore, in the `&publisher;` example, the `&publisher;` is not saved, but instead, an **XmlText** node is created.</span></span>
