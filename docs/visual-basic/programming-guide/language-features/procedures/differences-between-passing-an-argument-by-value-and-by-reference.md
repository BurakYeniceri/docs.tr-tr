<span data-ttu-id="b9f89-103">Bir yordama bağımsız değişkenlerden biri veya daha fazla geçirdiğinizde, her bir bağımsız değişkeni çağıran kodu temel programlama öğesi karşılık gelir.</span><span class="sxs-lookup"><span data-stu-id="b9f89-103">When you pass one or more arguments to a procedure, each argument corresponds to an underlying programming element in the calling code.</span></span> <span data-ttu-id="b9f89-104">Bu temel alınan öğenin değerini ya da bir başvuru geçirebilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="b9f89-104">You can pass either the value of this underlying element, or a reference to it.</span></span> <span data-ttu-id="b9f89-105">Bu olarak bilinir *mekanizması geçirme*.</span><span class="sxs-lookup"><span data-stu-id="b9f89-105">This is known as the *passing mechanism*.</span></span>  
  
## <a name="passing-by-value"></a><span data-ttu-id="b9f89-106">Değere göre geçirme</span><span class="sxs-lookup"><span data-stu-id="b9f89-106">Passing by Value</span></span>  
 <span data-ttu-id="b9f89-107">Bağımsız değişken geçirmek *değeriyle* belirterek [ByVal](../../../../visual-basic/language-reference/modifiers/byval.md) yordamı tanımında karşılık gelen bir parametre için anahtar sözcük.</span><span class="sxs-lookup"><span data-stu-id="b9f89-107">You pass an argument *by value* by specifying the [ByVal](../../../../visual-basic/language-reference/modifiers/byval.md) keyword for the corresponding parameter in the procedure definition.</span></span> <span data-ttu-id="b9f89-108">Bu mekanizma, geçirme kullandığınızda [!INCLUDE[vbprvb](~/includes/vbprvb-md.md)] yordamı yerel bir değişkende içine temel programlama öğesinin değeri kopyalar.</span><span class="sxs-lookup"><span data-stu-id="b9f89-108">When you use this passing mechanism, [!INCLUDE[vbprvb](~/includes/vbprvb-md.md)] copies the value of the underlying programming element into a local variable in the procedure.</span></span> <span data-ttu-id="b9f89-109">Yordamı kod, çağıran kodu temel öğesine herhangi bir erişim yok.</span><span class="sxs-lookup"><span data-stu-id="b9f89-109">The procedure code does not have any access to the underlying element in the calling code.</span></span>  
  
## <a name="passing-by-reference"></a><span data-ttu-id="b9f89-110">Başvuruya göre geçirme</span><span class="sxs-lookup"><span data-stu-id="b9f89-110">Passing by Reference</span></span>  
 <span data-ttu-id="b9f89-111">Bağımsız değişken geçirmek *başvuruya* belirterek [ByRef](../../../../visual-basic/language-reference/modifiers/byref.md) yordamı tanımında karşılık gelen bir parametre için anahtar sözcük.</span><span class="sxs-lookup"><span data-stu-id="b9f89-111">You pass an argument *by reference* by specifying the [ByRef](../../../../visual-basic/language-reference/modifiers/byref.md) keyword for the corresponding parameter in the procedure definition.</span></span> <span data-ttu-id="b9f89-112">Bu mekanizma, geçirme kullandığınızda [!INCLUDE[vbprvb](~/includes/vbprvb-md.md)] yordamı çağıran kodu temel programlama öğesi için doğrudan bir başvuru sağlar.</span><span class="sxs-lookup"><span data-stu-id="b9f89-112">When you use this passing mechanism, [!INCLUDE[vbprvb](~/includes/vbprvb-md.md)] gives the procedure a direct reference to the underlying programming element in the calling code.</span></span>  
  
## <a name="passing-mechanism-and-element-type"></a><span data-ttu-id="b9f89-113">Geçirme mekanizması ve öğe türü</span><span class="sxs-lookup"><span data-stu-id="b9f89-113">Passing Mechanism and Element Type</span></span>  
 <span data-ttu-id="b9f89-114">Mekanizması geçirme seçeneğiniz sınıflandırması, temel alınan öğe türü ile aynı değil.</span><span class="sxs-lookup"><span data-stu-id="b9f89-114">The choice of passing mechanism is not the same as the classification of the underlying element type.</span></span> <span data-ttu-id="b9f89-115">Değere veya başvuruya göre geçirme ne başvurur [!INCLUDE[vbprvb](~/includes/vbprvb-md.md)] yordamı kodu sağlar.</span><span class="sxs-lookup"><span data-stu-id="b9f89-115">Passing by value or by reference refers to what [!INCLUDE[vbprvb](~/includes/vbprvb-md.md)] supplies to the procedure code.</span></span> <span data-ttu-id="b9f89-116">Bir değer veya başvuru türü bir programlama öğesi bellekte nasıl depolandığını için ifade eder.</span><span class="sxs-lookup"><span data-stu-id="b9f89-116">A value type or reference type refers to how a programming element is stored in memory.</span></span>  
  
 <span data-ttu-id="b9f89-117">Ancak, öğe türü ve geçirme mekanizması birbirleriyle ilişkilidir.</span><span class="sxs-lookup"><span data-stu-id="b9f89-117">However, the passing mechanism and element type are interrelated.</span></span> <span data-ttu-id="b9f89-118">Bir başvuru türü değeri bellek verilerde başka bir yerde bir işaretçidir.</span><span class="sxs-lookup"><span data-stu-id="b9f89-118">The value of a reference type is a pointer to the data elsewhere in memory.</span></span> <span data-ttu-id="b9f89-119">Temel alınan öğe erişemiyor olsa bile bu değere göre bir başvuru türü geçirdiğinizde, yordam kodu temel alınan öğenin veri için bir işaretçi olduğu anlamına gelir.</span><span class="sxs-lookup"><span data-stu-id="b9f89-119">This means that when you pass a reference type by value, the procedure code has a pointer to the underlying element's data, even though it cannot access the underlying element itself.</span></span> <span data-ttu-id="b9f89-120">Örneğin, bir dizi değişkeni öğesiyse yordamı kod değişken erişimi yok, ancak dizi üyeleri erişebilir.</span><span class="sxs-lookup"><span data-stu-id="b9f89-120">For example, if the element is an array variable, the procedure code does not have access to the variable itself, but it can access the array members.</span></span>
