# <a name="adopting-windows-communication-foundation"></a><span data-ttu-id="58134-102">Windows Communication Foundation'ı Benimseme</span><span class="sxs-lookup"><span data-stu-id="58134-102">Adopting Windows Communication Foundation</span></span>
<span data-ttu-id="58134-103">ASP.NET kullanılarak geliştirilmiş uygulamalara sürdürmek devam ederken yeni geliştirme projeleri için Windows Communication Foundation (WCF) kullanmayı seçebilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="58134-103">You can choose to use Windows Communication Foundation (WCF) for new development, while continuing to maintain existing applications developed using ASP.NET.</span></span> <span data-ttu-id="58134-104">WCF herhangi bir senaryoda .NET Framework ile oluşturulan uygulamalar ile iletişimi kolaylaştırmanın için en uygun seçenek amaçlandığından, çok çeşitli şekilde yazılım iletişim sorunları çözmek için standart bir araç olarak ASP.NET görebilir uygulanamaz.</span><span class="sxs-lookup"><span data-stu-id="58134-104">Because WCF is intended to be the most suitable choice for facilitating communication with applications built with the .NET Framework in any scenario, it can serve as a standard tool for solving a wide variety of software communications problems in a way that ASP.NET cannot.</span></span>  
  
 <span data-ttu-id="58134-105">Yeni WCF uygulamaları mevcut ASP.NET Web Hizmetleri ile aynı makinede dağıtılabilir.</span><span class="sxs-lookup"><span data-stu-id="58134-105">New WCF applications can be deployed on the same machines as existing ASP.NET Web services.</span></span> <span data-ttu-id="58134-106">Mevcut ASP.NET Web Hizmetleri, .NET Framework sürüm 2.0 önce bir sürümünü kullanıyorsanız, seçmeli olarak .NET Framework 2.0 yeni WCF uygulamaları barındırılacak şekilde olduğu IIS uygulama dağıtmak için ASP.NET IIS Kayıt aracını kullanabilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="58134-106">If the existing ASP.NET Web services use a version of the .NET Framework prior to version 2.0, then you can use the ASP.NET IIS Registration Tool to selectively deploy the .NET Framework 2.0 to IIS applications in which new WCF applications are to be hosted.</span></span> <span data-ttu-id="58134-107">Bu aracı adresinde belgelenen [ASP.NET IIS Kayıt Aracı (Aspnet_regiis.exe)](http://go.microsoft.com/fwlink/?LinkId=94687), ve bir kullanıcı arabirimi yerleşik IIS 6.0 yönetim konsoluna sahip.</span><span class="sxs-lookup"><span data-stu-id="58134-107">That tool is documented at [ASP.NET IIS Registration Tool (Aspnet_regiis.exe)](http://go.microsoft.com/fwlink/?LinkId=94687), and has a user interface built into the IIS 6.0 management console.</span></span>  
  
 <span data-ttu-id="58134-108">WCF IIS'ndaki mevcut ASP.NET Web hizmeti uygulama ASP.NET uyumluluk modunda çalışacak şekilde yapılandırılmış WCF hizmetleri ekleyerek mevcut ASP.NET Web Hizmetleri için yeni özellikler eklemek için kullanılabilir.</span><span class="sxs-lookup"><span data-stu-id="58134-108">WCF can be used to add new features to existing ASP.NET Web services by adding WCF services configured to run in ASP.NET compatibility mode to existing ASP.NET Web service applications in IIS.</span></span> <span data-ttu-id="58134-109">ASP.NET uyumluluğu modunu nedeniyle yeni WCF hizmetleri için kod erişebilir ve önceden var olan ASP.NET kodu olarak aynı uygulama durum bilgilerini kullanarak güncelleştirme <xref:System.Web.HttpContext> sınıfı.</span><span class="sxs-lookup"><span data-stu-id="58134-109">Because of ASP.NET compatibility mode, the code for the new WCF services can access and update the same application state information as the pre-existing ASP.NET code, by using the <xref:System.Web.HttpContext> class.</span></span> <span data-ttu-id="58134-110">Uygulamaları, aynı sınıf kitaplıkları da paylaşabilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="58134-110">The applications can also share the same class libraries.</span></span>  
  
 <span data-ttu-id="58134-111">WCF istemcileri, ASP.NET Web hizmetlerini kullanabilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="58134-111">WCF clients can use ASP.NET Web services.</span></span> <span data-ttu-id="58134-112">İle yapılandırılmış WCF hizmetleri <xref:System.ServiceModel.BasicHttpBinding> ASP.NET Web hizmeti istemciler tarafından kullanılabilir.</span><span class="sxs-lookup"><span data-stu-id="58134-112">WCF services that are configured with the <xref:System.ServiceModel.BasicHttpBinding> can be used by ASP.NET Web service clients.</span></span> <span data-ttu-id="58134-113">ASP.NET Web Hizmetleri ile WCF uygulamaları birlikte bulunabilir ve WCF bile mevcut ASP.NET Web hizmetlerini özellikleri eklemek için kullanılabilir.</span><span class="sxs-lookup"><span data-stu-id="58134-113">ASP.NET Web services can co-exist with WCF applications, and WCF can even be used to add features to existing ASP.NET Web services.</span></span> <span data-ttu-id="58134-114">Bu şekilde, WCF ve ASP.NET Web Hizmetleri birlikte kullanılabilir mi tümünün verilen, yalnızca WCF ve ASP.NET Web Hizmetleri tarafından sağlanan özellikleri gerekiyorsa ASP.NET Web hizmetlerini WCF'ye taşıma isteyebilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="58134-114">Given all of these ways in which WCF and ASP.NET Web services can be used together, you may want to migrate ASP.NET Web services to WCF only if you require features that are provided by WCF and not ASP.NET Web services.</span></span>  
  
 <span data-ttu-id="58134-115">Gerekli olduğu bile bazı durumlarda, bir teknoloji geçirme koddan diğerine nadiren doğru yaklaşım olduğunu dikkatlice düşünün.</span><span class="sxs-lookup"><span data-stu-id="58134-115">Even in the few cases where it is necessary, carefully consider that migrating code from one technology to another is seldom the correct approach.</span></span> <span data-ttu-id="58134-116">Yeni teknoloji benimsenmesi nedeni önceki teknolojisiyle karşılanamıyor yeni gereksinimlerini karşılamak için ve bu durumda, yapmak için doğru olanı yeni Genişletilmiş gereksinimleri karşılaması için yeni bir çözüm tasarımı için.</span><span class="sxs-lookup"><span data-stu-id="58134-116">The reason for adopting the new technology is to meet new requirements that cannot be met with the earlier technology, and in that case, the correct thing to do is to design a new solution to meet the newly-expanded set of requirements.</span></span> <span data-ttu-id="58134-117">Bu sistem tasarlandığından beri varolan sistem deneyiminizi ve wisdom yeni tasarım avantajları elde edilen.</span><span class="sxs-lookup"><span data-stu-id="58134-117">The new design benefits from your experience with the existing system and from wisdom gained since that system was designed.</span></span> <span data-ttu-id="58134-118">Yeni Tasarım, yeni bir platformda eski tasarım yeniden oluşturma yerine yeni teknolojilerden tam özelliklerini de kullanabilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="58134-118">The new design can also use the full capabilities of the new technologies rather than reproducing the old design on the new platform.</span></span> <span data-ttu-id="58134-119">Prototipi oluşturulurken anahtar öğeleri yeni tasarım, sonra bu kodu yeniden kullanma yeni bir içinde mevcut sisteminden daha kolay olur.</span><span class="sxs-lookup"><span data-stu-id="58134-119">After prototyping key elements of the new design, it becomes easier to re-use code from the existing system within the new one.</span></span>  
  
 <span data-ttu-id="58134-120">ASP.NET Web hizmetlerini WCF'ye taşıma burada birkaç durumda olduğu için doğru çözüm, aşağıdaki bölümde nasıl devam etmek bazı yönergeler sağlanmaktadır.</span><span class="sxs-lookup"><span data-stu-id="58134-120">For the few cases where porting from ASP.NET Web services to WCF is the correct solution, the following section provides some guidance on how to proceed.</span></span> <span data-ttu-id="58134-121">Öneri Hizmetleri'ni geçirmek nasıl ve istemcileri geçirmek nasıl yoktur.</span><span class="sxs-lookup"><span data-stu-id="58134-121">There is advice on how to migrate services, and how to migrate clients.</span></span>  
  
 <span data-ttu-id="58134-122">Mevcut ASP.NET Web hizmetlerini WCF'ye taşıma konusunda tam bir çözümleme için lütfen bkz [ASP.NET Web Hizmetleri ve Windows Communication Foundation](http://go.microsoft.com/fwlink/?LinkID=71761).</span><span class="sxs-lookup"><span data-stu-id="58134-122">For a complete analysis on how to migrate existing ASP.NET Web Services to WCF please see [ASP.NET Web Services and the Windows Communication Foundation](http://go.microsoft.com/fwlink/?LinkID=71761).</span></span> <span data-ttu-id="58134-123">Bu bölümde meta verilerden uyumlu bir WCF hizmeti, ASP.NET Web hizmeti uygulama ve ASP.NET Web hizmeti ve istemci kodunu WCF'ye taşıma anlatılmaktadır.</span><span class="sxs-lookup"><span data-stu-id="58134-123">This section describes how to implement a compliant WCF service from the metadata for you ASP.NET Web Service, and how to migrate ASP.NET Web service and client code to WCF.</span></span>
