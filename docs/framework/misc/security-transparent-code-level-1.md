## <a name="the-level-1-transparency-model"></a><span data-ttu-id="3e9d9-114">Düzey 1 saydamlık modeli</span><span class="sxs-lookup"><span data-stu-id="3e9d9-114">The Level 1 Transparency Model</span></span>  
 <span data-ttu-id="3e9d9-115">Düzey 1 saydamlık kullandığınızda, güvenlik saydam, güvenlik açısından güvenli-kritik ve güvenlik açısından kritik yöntemlerin içine kodu ayıran bir güvenlik modeli kullanıyor.</span><span class="sxs-lookup"><span data-stu-id="3e9d9-115">When you use Level 1 transparency, you are using a security model that separates code into security-transparent, security-safe-critical, and security-critical methods.</span></span>  
  
 <span data-ttu-id="3e9d9-116">Tüm bir derleme, bir derlemede bazı sınıflar ya da bir sınıf güvenliği saydam olarak bazı yöntemleri işaretleyebilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="3e9d9-116">You can mark a whole assembly, some classes in an assembly, or some methods in a class as security-transparent.</span></span> <span data-ttu-id="3e9d9-117">Güvenliği saydam kod ayrıcalık yükseltme yapamazsınız.</span><span class="sxs-lookup"><span data-stu-id="3e9d9-117">Security-transparent code cannot elevate privileges.</span></span> <span data-ttu-id="3e9d9-118">Bu kısıtlama üç sonuçları vardır:</span><span class="sxs-lookup"><span data-stu-id="3e9d9-118">This restriction has three consequences:</span></span>

-   <span data-ttu-id="3e9d9-120">Güvenliği saydam kod tarafından karşılanması herhangi bir bağlantı isteği tam isteğe bağlı olur.</span><span class="sxs-lookup"><span data-stu-id="3e9d9-120">Any link demand that would be satisfied by security-transparent code becomes a full demand.</span></span>  
  
-   <span data-ttu-id="3e9d9-121">Güvenliği saydam kod içinde yürütülmelidir güvenli olmayan (doğrulanamaz) kod için tam talep neden <xref:System.Security.Permissions.SecurityPermissionFlag.UnmanagedCode> güvenlik izni.</span><span class="sxs-lookup"><span data-stu-id="3e9d9-121">Any unsafe (unverifiable) code that must execute in security-transparent code causes a full demand for the <xref:System.Security.Permissions.SecurityPermissionFlag.UnmanagedCode> security permission.</span></span>

## <a name="security-transparency-examples"></a><span data-ttu-id="3e9d9-181">Güvenlik saydamlık örnekleri</span><span class="sxs-lookup"><span data-stu-id="3e9d9-181">Security Transparency Examples</span></span>

```

```
