## <a name="manners-of-execution"></a><span data-ttu-id="a618b-108">Yolla yürütme</span><span class="sxs-lookup"><span data-stu-id="a618b-108">Manners of Execution</span></span>  
  
### <a name="immediate"></a><span data-ttu-id="a618b-109">Hemen</span><span class="sxs-lookup"><span data-stu-id="a618b-109">Immediate</span></span>  
 <span data-ttu-id="a618b-110">Hemen bir yürütme veri kaynağı okuyun ve işlem kodda bir noktada sorgu burada bildirilmiş gerçekleştirilir anlamına gelir.</span><span class="sxs-lookup"><span data-stu-id="a618b-110">Immediate execution means that the data source is read and the operation is performed at the point in the code where the query is declared.</span></span> <span data-ttu-id="a618b-111">Numaralandırılabilir olmayan, tek bir sonuç tüm standart sorgu işleçleri hemen yürütün.</span><span class="sxs-lookup"><span data-stu-id="a618b-111">All the standard query operators that return a single, non-enumerable result execute immediately.</span></span>  
  
### <a name="deferred"></a><span data-ttu-id="a618b-112">Ertelenmiş</span><span class="sxs-lookup"><span data-stu-id="a618b-112">Deferred</span></span>  
 <span data-ttu-id="a618b-113">Ertelenmiş yürütme işlemi kodda bir noktada sorgu burada bildirilmiş gerçekleştirildiğini değil anlamına gelir.</span><span class="sxs-lookup"><span data-stu-id="a618b-113">Deferred execution means that the operation is not performed at the point in the code where the query is declared.</span></span> <span data-ttu-id="a618b-114">Yalnızca sorgu değişkeni, örneğin kullanarak numaralandırılır işlemi gerçekleştirilir bir `foreach` deyimi.</span><span class="sxs-lookup"><span data-stu-id="a618b-114">The operation is performed only when the query variable is enumerated, for example by using a `foreach` statement.</span></span> <span data-ttu-id="a618b-115">Bu sorgu, sorgu zaman tanımlanan yerine yürütüldüğünde sorgu yürütülürken sonuçlarını'nın veri kaynağını içeriğine bağlı anlamına gelir.</span><span class="sxs-lookup"><span data-stu-id="a618b-115">This means that the results of executing the query depend on the contents of the data source when the query is executed rather than when the query is defined.</span></span> <span data-ttu-id="a618b-116">Sorgu değişkeni birden çok kez listelenebilir, sonuçları her zaman farklı olabilir.</span><span class="sxs-lookup"><span data-stu-id="a618b-116">If the query variable is enumerated multiple times, the results might differ every time.</span></span> <span data-ttu-id="a618b-117">Dönüş türü olan neredeyse tüm standart sorgu işleçleri <xref:System.Collections.Generic.IEnumerable%601> veya <xref:System.Linq.IOrderedEnumerable%601> ertelenmiş bir biçimde yürütün.</span><span class="sxs-lookup"><span data-stu-id="a618b-117">Almost all the standard query operators whose return type is <xref:System.Collections.Generic.IEnumerable%601> or <xref:System.Linq.IOrderedEnumerable%601> execute in a deferred manner.</span></span>  
  
 <span data-ttu-id="a618b-118">Ertelenmiş yürütme kullanmak sorgu işleçleri ayrıca akış veya akış dışı olarak sınıflandırılabilir.</span><span class="sxs-lookup"><span data-stu-id="a618b-118">Query operators that use deferred execution can be additionally classified as streaming or non-streaming.</span></span>  
  
#### <a name="streaming"></a><span data-ttu-id="a618b-119">Akış</span><span class="sxs-lookup"><span data-stu-id="a618b-119">Streaming</span></span>  
 <span data-ttu-id="a618b-120">Akış işleçleri öğeleri verim önce tüm kaynak verileri okumak zorunda değildir.</span><span class="sxs-lookup"><span data-stu-id="a618b-120">Streaming operators do not have to read all the source data before they yield elements.</span></span> <span data-ttu-id="a618b-121">Salt okunur ve uygunsa öğesi verir yürütme zaman bir akış işleci her source öğesi kendi işlemi gerçekleştirir.</span><span class="sxs-lookup"><span data-stu-id="a618b-121">At the time of execution, a streaming operator performs its operation on each source element as it is read and yields the element if appropriate.</span></span> <span data-ttu-id="a618b-122">Bir akış işleci bir sonuç öğesi üretilen kadar kaynak öğelerin okumaya devam eder.</span><span class="sxs-lookup"><span data-stu-id="a618b-122">A streaming operator continues to read source elements until a result element can be produced.</span></span> <span data-ttu-id="a618b-123">Başka bir deyişle, bir sonuç öğesi oluşturmak için birden fazla kaynak öğesi okuma.</span><span class="sxs-lookup"><span data-stu-id="a618b-123">This means that more than one source element might be read to produce one result element.</span></span>  
  
#### <a name="non-streaming"></a><span data-ttu-id="a618b-124">Akış dışı</span><span class="sxs-lookup"><span data-stu-id="a618b-124">Non-Streaming</span></span>  
 <span data-ttu-id="a618b-125">Bir sonuç öğesi sağlayabilen önce olmayan akış işleçler tüm kaynak verileri okumalısınız.</span><span class="sxs-lookup"><span data-stu-id="a618b-125">Non-streaming operators must read all the source data before they can yield a result element.</span></span> <span data-ttu-id="a618b-126">Sıralama veya gruplama gibi işlemler bu kategoriye girer.</span><span class="sxs-lookup"><span data-stu-id="a618b-126">Operations such as sorting or grouping fall into this category.</span></span> <span data-ttu-id="a618b-127">Yürütme zaman, akış dışı sorgu işleçleri tüm kaynak verileri okumak, bir veri yapısında put, işlemi gerçekleştirmek ve sonuçta elde edilen öğeleri verim.</span><span class="sxs-lookup"><span data-stu-id="a618b-127">At the time of execution, non-streaming query operators read all the source data, put it into a data structure, perform the operation, and yield the resulting elements.</span></span>
