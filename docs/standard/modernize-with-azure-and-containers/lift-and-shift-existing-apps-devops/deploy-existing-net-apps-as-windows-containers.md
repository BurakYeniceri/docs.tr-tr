## <a name="windows-container-types"></a><span data-ttu-id="46197-209">Windows kapsayıcı türleri</span><span class="sxs-lookup"><span data-stu-id="46197-209">Windows container types</span></span>

<span data-ttu-id="46197-210">Linux kapsayıcıları gibi Windows Server kapsayıcıları, Docker altyapısına kullanılarak yönetilir.</span><span class="sxs-lookup"><span data-stu-id="46197-210">Like Linux containers, Windows Server containers are managed by using Docker Engine.</span></span> <span data-ttu-id="46197-211">Linux kapsayıcıları aksine Windows kapsayıcıları iki farklı kapsayıcı türler veya kez Windows Server kapsayıcıları ve Hyper-V yalıtım çalıştırın.</span><span class="sxs-lookup"><span data-stu-id="46197-211">Unlike Linux containers, Windows containers include two different container types, or run times-Windows Server containers and Hyper-V isolation.</span></span>

<span data-ttu-id="46197-212">**Windows Server kapsayıcıları**: işlem ve ad alanı yalıtım teknolojisi aracılığıyla uygulama yalıtımı sağlar.</span><span class="sxs-lookup"><span data-stu-id="46197-212">**Windows Server containers**: Provides application isolation through process and namespace isolation technology.</span></span> <span data-ttu-id="46197-213">Bir Windows Server kapsayıcı bir çekirdek kapsayıcı ana bilgisayar ve ana bilgisayarda çalışan tüm kapsayıcıları ile paylaşır.</span><span class="sxs-lookup"><span data-stu-id="46197-213">A Windows Server container shares a kernel with the container host and all containers that are running on the host.</span></span> <span data-ttu-id="46197-214">Bu kapsayıcıların bir saldırgan güvenlik sınırı sağlamaz ve güvenilmeyen kod yalıtmak için kullanılmamalıdır.</span><span class="sxs-lookup"><span data-stu-id="46197-214">These containers do not provide a hostile security boundary and should not be used to isolate untrusted code.</span></span> <span data-ttu-id="46197-215">Paylaşılan çekirdek alanı nedeniyle bu kapsayıcılar aynı çekirdek sürümü ve yapılandırma gerektirir.</span><span class="sxs-lookup"><span data-stu-id="46197-215">Because of the shared kernel space, these containers require the same kernel version and configuration.</span></span>

<span data-ttu-id="46197-216">**Hyper-V yalıtım**: yüksek oranda iyileştirilmiş bir VM üzerinde her kapsayıcı çalıştırarak Windows Server kapsayıcıları tarafından sağlanan yalıtımı genişletir.</span><span class="sxs-lookup"><span data-stu-id="46197-216">**Hyper-V isolation**: Expands on the isolation provided by Windows Server Containers by running each container on a highly optimized VM.</span></span> <span data-ttu-id="46197-217">Bu yapılandırmada, aynı ana bilgisayardaki diğer kapsayıcılarla kapsayıcı konağının çekirdek paylaşılmaz.</span><span class="sxs-lookup"><span data-stu-id="46197-217">In this configuration, the kernel of the container host is not shared with other containers on the same host.</span></span> <span data-ttu-id="46197-218">Bu kapsayıcılar, saldırgan çok müşterili, bir VM ile aynı güvenlik çıkışların barındırmak için tasarlanmıştır.</span><span class="sxs-lookup"><span data-stu-id="46197-218">These containers are designed for hostile multitenant hosting, with the same security assurances of a VM.</span></span> <span data-ttu-id="46197-219">Bu kapsayıcılar konak veya konak üzerindeki diğer kapsayıcıları çekirdek paylaşmayın olduğundan farklı sürümleri ve yapılandırmalarla (desteklenen sürümler) tekrar çalıştırabilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="46197-219">Because these containers don't share the kernel with the host or other containers on the host, they can run kernels with different versions and configurations (with supported versions).</span></span> <span data-ttu-id="46197-220">Örneğin, tüm Windows kapsayıcıları Windows 10 Windows Server çekirdek sürümü ve yapılandırmasını kullanmak için Hyper-V yalıtımını kullanın.</span><span class="sxs-lookup"><span data-stu-id="46197-220">For example, all Windows containers on Windows 10 use Hyper-V isolation to utilize the Windows Server kernel version and configuration.</span></span>

<span data-ttu-id="46197-221">Bir kapsayıcı ile veya Hyper-V yalıtım olmadan Windows üzerinde çalışan çalışma zamanında bir karardır.</span><span class="sxs-lookup"><span data-stu-id="46197-221">Running a container on Windows with or without Hyper-V isolation is a run-time decision.</span></span> <span data-ttu-id="46197-222">Başlangıçta Hyper-V yalıtım ile kapsayıcı oluşturulacağını seçin ve çalıştırmak Windows Server kapsayıcı olarak bunun yerine çalışma zamanında seçin.</span><span class="sxs-lookup"><span data-stu-id="46197-222">You might choose to create the container with Hyper-V isolation initially, and at run time, choose to run it as a Windows Server container instead.</span></span>
