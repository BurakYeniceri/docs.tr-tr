### <a name="defining-an-event-bus-interface"></a><span data-ttu-id="87782-168">Bir olay veri yolu arabirim tanımlama</span><span class="sxs-lookup"><span data-stu-id="87782-168">Defining an event bus interface</span></span>

<span data-ttu-id="87782-169">Bazı uygulama kodu olay veri yolu arabirimin ve araştırma amacıyla olası uygulamaları ile başlayalım tıklatın.</span><span class="sxs-lookup"><span data-stu-id="87782-169">Let’s start with some implementation code for the event bus interface and possible implementations for exploration purposes.</span></span> <span data-ttu-id="87782-170">Arabirimi, genel ve basit arabirim olduğu gibi olması gerekir.</span><span class="sxs-lookup"><span data-stu-id="87782-170">The interface should be generic and straightforward, as in the following interface.</span></span>

```csharp
public interface IEventBus
{
    void Publish(IntegrationEvent @event);

    void Subscribe<T, TH>()
        where T : IntegrationEvent
        where TH : IIntegrationEventHandler<T>;

    void SubscribeDynamic<TH>(string eventName)
        where TH : IDynamicIntegrationEventHandler;

    void UnsubscribeDynamic<TH>(string eventName)
        where TH : IDynamicIntegrationEventHandler;

    void Unsubscribe<T, TH>()
        where TH : IIntegrationEventHandler<T>
        where T : IntegrationEvent;
}
```

<span data-ttu-id="87782-171">`Publish` Yöntem basittir.</span><span class="sxs-lookup"><span data-stu-id="87782-171">The `Publish` method is straightforward.</span></span> <span data-ttu-id="87782-172">Olay veri yolu, herhangi bir mikro hizmet ya da o olaya abone bile bir dış uygulamaya geçirilen tümleştirme etkinliği yayınlamak.</span><span class="sxs-lookup"><span data-stu-id="87782-172">The event bus will broadcast the integration event passed to it to any microservice, or even an external application, subscribed to that event.</span></span> <span data-ttu-id="87782-173">Bu yöntem olayın yayımlanması mikro hizmet tarafından kullanılır.</span><span class="sxs-lookup"><span data-stu-id="87782-173">This method is used by the microservice that is publishing the event.</span></span>

<span data-ttu-id="87782-174">`Subscribe` Yöntemleri (bağımsız değişkenler bağlı olarak çeşitli uygulamalar olabilir), olaylar almak istiyorsanız mikro hizmetler tarafından kullanılır.</span><span class="sxs-lookup"><span data-stu-id="87782-174">The `Subscribe` methods (you can have several implementations depending on the arguments) are used by the microservices that want to receive events.</span></span> <span data-ttu-id="87782-175">Bu yöntem, iki bağımsız değişkeni vardır.</span><span class="sxs-lookup"><span data-stu-id="87782-175">This method has two arguments.</span></span> <span data-ttu-id="87782-176">İlk abone olmak için tümleştirme olaydır (`IntegrationEvent`).</span><span class="sxs-lookup"><span data-stu-id="87782-176">The first is the integration event to subscribe to (`IntegrationEvent`).</span></span> <span data-ttu-id="87782-177">Tümleştirme olay işleyici (ya da geri arama yöntemi) adlı ikinci bağımsız değişken olan `IIntegrationEventHandler<T>`, alıcı mikro hizmet, bu tümleştirme olay iletisi aldığında çalıştırılacak.</span><span class="sxs-lookup"><span data-stu-id="87782-177">The second argument is the integration event handler (or callback method), named `IIntegrationEventHandler<T>`, to be executed when the receiver microservice gets that integration event message.</span></span>
