### <a name="connection-pool-blocking-period-for-azure-sql-databases-is-removed"></a>Bağlantı havuzu Azure SQL veritabanları için dönem engelleme kaldırıldı

|   |   |
|---|---|
|Ayrıntılar|.NET Framework 4.6.2 ile başlayarak, bağlantı için bilinen Azure SQL veritabanları için açık olan istekleri (*. database.windows.net, *. database.chinacloudapi.cn, *. database.usgovcloudapi.net, *. database.cloudapi.de), bağlantı havuzu engelleme süresi kaldırılır ve bağlantı açık hatalarını önbelleğe alınmaz. Bağlantı açık isteklerini yeniden deneme girişimleri hemen sonra geçici bağlantı hatalarını meydana gelir. Bu değişiklik, böylece bulut-etkinleştirilmiş uygulamaların performansını iyileştirme Azure SQL veritabanları için hemen denenmesi bağlantı açık denemesi sağlar. Diğer tüm bağlantı girişimleri için bağlantı havuzunu engelleme süresi zorlanacak devam eder.<p/>Bir uygulama, bir veritabanına bağlanırken bir geçici bağlantı hatası karşılaştığında çünkü bağlantı havuzu hata önbelleğe alır ve bunu 5 saniyede 1 için yeniden oluşturur .NET Framework 4.6.1 ve önceki sürümleri, bağlantı girişimi hızlı bir şekilde, denenemiyor dakika. Daha fazla bilgi için bkz: [SQL Server bağlantı havuzu (ADO.NET)](~/docs/framework/data/adonet/sql-server-connection-pooling.md). Bu bağlantılar genellikle genellikle gelen birkaç saniye içinde kurtarılan geçici hataları ile başarısız Azure SQL veritabanları için sorunlu davranıştır. Bağlantı havuzu engelleme özelliği, uygulama veritabanına kapsamlı bir dönem için veritabanının kullanılabilir olduğundan ve birkaç saniye içinde işlemek uygulamanın gerekir olsa bile bağlanamadığını anlamına gelir.|
|Öneri|Bu davranış istenmeyen ise, bağlantı havuzu engelleme süresi ayarlayarak yapılandırılabilir <xref:System.Data.SqlClient.SqlConnectionStringBuilder.PoolBlockingPeriod?displayProperty=name> .NET Framework 4.6.2 sunulan özelliği. Özelliğinin değeri bir üyesidir <xref:System.Data.SqlClient.PoolBlockingPeriod?displayProperty=name> numaralandırması üç değerden birini alabilir:<ul><li><xref:System.Data.SqlClient.PoolBlockingPeriod.AlwaysBlock></li><li><xref:System.Data.SqlClient.PoolBlockingPeriod.Auto></li><li><xref:System.Data.SqlClient.PoolBlockingPeriod.NeverBlock></li></ul>Önceki davranış ayarlayarak geri yüklenebilir <xref:System.Data.SqlClient.SqlConnectionStringBuilder.PoolBlockingPeriod?displayProperty=name> özelliğine <xref:System.Data.SqlClient.PoolBlockingPeriod.AlwaysBlock>.|
