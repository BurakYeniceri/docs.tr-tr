 <span data-ttu-id="35f94-108">İçinde *saf işlev dönüştürme*, çağrılan işlevler kümesi *saf işlevleri*, özgün formuna yapılandırılmış veri kümesi başka bir biçime dönüştürmek nasıl tanımlayın.</span><span class="sxs-lookup"><span data-stu-id="35f94-108">In *pure functional transformation*, a set of functions, called *pure functions*, define how to transform a set of structured data from its original form into another form.</span></span> <span data-ttu-id="35f94-109">"Saf" word işlevleri gösterir *birleştirilebilir*, olmalarını gerektirir:</span><span class="sxs-lookup"><span data-stu-id="35f94-109">The word "pure" indicates that the functions are *composable*, which requires that they are:</span></span>  
  
-   <span data-ttu-id="35f94-110">*Kendi içinde bulunan*, böylece bunlar kullanılabilir ücretsiz olarak sipariş ve entanglement veya program kalanıyla bağımlılıklarını olmadan düzenlenmeyecek.</span><span class="sxs-lookup"><span data-stu-id="35f94-110">*Self-contained*, so that they can be freely ordered and rearranged without entanglement or interdependencies with the rest of the program.</span></span> <span data-ttu-id="35f94-111">Saf dönüşümleri hiçbir bilgisi veya ortamlarına üzerinde etkisi vardır.</span><span class="sxs-lookup"><span data-stu-id="35f94-111">Pure transformations have no knowledge of or effect upon their environment.</span></span> <span data-ttu-id="35f94-112">Diğer bir deyişle, dönüşümünde işlevleri yok *yan etkileri*.</span><span class="sxs-lookup"><span data-stu-id="35f94-112">That is, the functions used in the transformation have no *side effects*.</span></span>
