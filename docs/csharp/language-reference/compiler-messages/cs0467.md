# <a name="compiler-warning-level-2-cs0467"></a><span data-ttu-id="3f983-102">Derleyici Uyarısı (düzey 2) CS0467</span><span class="sxs-lookup"><span data-stu-id="3f983-102">Compiler Warning (level 2) CS0467</span></span>
<span data-ttu-id="3f983-103">Yöntem olmayan 'yöntemi olmayan' yöntemi 'yöntemi' arasındaki belirsizlik. Yöntem grubunu kullanmak.</span><span class="sxs-lookup"><span data-stu-id="3f983-103">Ambiguity between method 'method' and non-method 'non-method'.Using method group.</span></span>  
  
 <span data-ttu-id="3f983-104">Belirsizliği hataya neden aynı imzaya sahip farklı arabirimlerinden devralınan üyeleri.</span><span class="sxs-lookup"><span data-stu-id="3f983-104">Inherited members from different interfaces that have the same signature  cause an ambiguity error.</span></span>  
  
## <a name="example"></a><span data-ttu-id="3f983-105">Örnek</span><span class="sxs-lookup"><span data-stu-id="3f983-105">Example</span></span>  
 <span data-ttu-id="3f983-106">Aşağıdaki örnek CS0467 oluşturur.</span><span class="sxs-lookup"><span data-stu-id="3f983-106">The following example generates CS0467.</span></span>  
  
```csharp  
// CS0467.cs  
interface IList   
{  
    int Count { get; set; }  
}  
  
interface ICounter  
{  
    void Count(int i);  
}  
  
interface IListCounter : IList, ICounter {}  
  
class Driver   
{  
    void Test(IListCounter x)  
    {  
        // The following line causes the warning. The assignment also  
        // causes an error because you can't assign a value to a method.  
        x.Count = 1;  
        x.Count(3);     
        // To resolve the warning, you can change the name of the method or   
        // the property.  
  
        // You also can disambiguate by specifying IList or ICounter.  
        ((IList)x).Count = 1;  
        ((ICounter)x).Count(3);  
    }  
  
    static void Main()   
    {  
    }  
}  
```
