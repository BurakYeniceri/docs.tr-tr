# <a name="how-to-make-thread-safe-calls-to-windows-forms-controls"></a>Nasıl yapılır: Windows Forms Denetimlerine İş Parçacığı Güvenli Aramalar Yapma
Kullanırsanız, çok iş parçacıklı Windows Forms uygulamalarınızın performansını artırmak için bir iş parçacığı açısından güvenli şekilde denetimlerinizi çağrı yapmak emin olmanız gerekir.  
  
 Windows Forms denetimleri için erişim kendiliğinden iş parçacığı içinde korumalı değil. İki varsa veya daha fazla iş parçacığı bir denetim durumu düzenleme, Denetim tutarsız bir duruma zorlaması mümkündür. Yarış durumları ve kilitlenmeleri gibi diğer iş parçacığı ile ilgili hatalar mümkündür. Erişim denetimleri için bir iş parçacığı açısından güvenli şekilde gerçekleştirilir emin olmak önemlidir.  
  
 Güvenli olmayan bir iş parçacığı denetimi kullanılmadan oluşturulmuş bir dışında bir denetim çağırmanıza <xref:System.Windows.Forms.Control.Invoke%2A> yöntemi. İş parçacığı güvenli olmayan bir örnek verilmiştir.  
  
```csharp  
// This event handler creates a thread that calls a   
// Windows Forms control in an unsafe way.  
private void setTextUnsafeBtn_Click(  
    object sender,   
    EventArgs e)  
{  
    this.demoThread =   
        new Thread(new ThreadStart(this.ThreadProcUnsafe));  
  
    this.demoThread.Start();  
}  
  
// This method is executed on the worker thread and makes  
// an unsafe call on the TextBox control.  
private void ThreadProcUnsafe()  
{  
    this.textBox1.Text = "This text was set unsafely.";  
}  
```  
  
```vb  
' This event handler creates a thread that calls a   
' Windows Forms control in an unsafe way.  
 Private Sub setTextUnsafeBtn_Click( _  
 ByVal sender As Object, _  
 ByVal e As EventArgs) Handles setTextUnsafeBtn.Click  
  
     Me.demoThread = New Thread( _  
     New ThreadStart(AddressOf Me.ThreadProcUnsafe))  
  
     Me.demoThread.Start()  
 End Sub  
  
' This method is executed on the worker thread and makes  
' an unsafe call on the TextBox control.  
Private Sub ThreadProcUnsafe()  
   Me.textBox1.Text = "This text was set unsafely."  
End Sub  
```  
  

 [!INCLUDE[dnprdnshort](../../../../includes/dnprdnshort-md.md)] Denetimlerinizi iş parçacığı güvenli olmayan bir şekilde erişirken, algılamanıza yardımcı olur. Ne zaman hata ayıklayıcısı'ndaki uygulamanızı çalıştıran ve bir denetim oluşturulan dışında bir iş parçacığı denetimi, hata ayıklayıcı başlatır çağırmayı dener bir <xref:System.InvalidOperationException> iletiyle "Denetim *denetim adı* erişilen bir iş parçacığı oluşturulduğu iş parçacığı dışında."  
  
 Bu özel durum, çalışma zamanında hata ayıklama sırasında ve bazı koşullarda güvenilir bir şekilde gerçekleşir. İle yazdığınız uygulamaları ayıklarken bu özel durumu görebilirsiniz [!INCLUDE[dnprdnshort](../../../../includes/dnprdnshort-md.md)] öncesinde [!INCLUDE[dnprdnext](../../../../includes/dnprdnext-md.md)]. Gördüğünüzde, ancak ayarlayarak devre dışı bırakabilirsiniz, bu sorunu gidermek için kesinlikle önerilir <xref:System.Windows.Forms.Control.CheckForIllegalCrossThreadCalls%2A> özelliğine `false`. Bu denetiminiz altında Visual Studio .NET 2003 çalışır gibi çalışmasına neden olur ve [!INCLUDE[net_v11_short](../../../../includes/net-v11-short-md.md)].  
  
> [!NOTE]
>  ActiveX denetimleri form üzerinde kullanıyorsanız, iş parçacıkları arası alabilirsiniz <xref:System.InvalidOperationException> hata ayıklayıcı altında çalıştırdığınızda. Bu durumda, ActiveX denetimini desteklemediği çoklu iş parçacığı kullanımı. Windows Forms ActiveX denetimlerini kullanma hakkında daha fazla bilgi için bkz: [Windows Forms ve yönetilmeyen uygulamalar](../../../../docs/framework/winforms/advanced/windows-forms-and-unmanaged-applications.md). Visual Studio kullanıyorsanız, Visual Studio barındırma işlemi devre dışı bırakarak bu özel durumun engellemek, bkz: [nasıl yapılır: barındırma sürecini devre dışı bırakma](/visualstudio/ide/how-to-disable-the-hosting-process).  
  
## <a name="making-thread-safe-calls-to-windows-forms-controls"></a>İş parçacığı aramalarına Windows Forms denetimleri  
  
#### <a name="to-make-a-thread-safe-call-to-a-windows-forms-control"></a>Bir Windows Forms denetimi için iş parçacığı arama yapmak için  
  
1.  Denetimin sorgu <xref:System.Windows.Forms.Control.InvokeRequired%2A> özelliği.  
  
2.  Varsa <xref:System.Windows.Forms.Control.InvokeRequired%2A> döndürür `true`, çağrı <xref:System.Windows.Forms.Control.Invoke%2A> gerçek çağrıda denetlemek için bir temsilci ile.  
  
3.  Varsa <xref:System.Windows.Forms.Control.InvokeRequired%2A> döndürür `false`, Denetim doğrudan çağırabilir.  
  
 Aşağıdaki kod örneğinde bir iş parçacığı araması uygulanan `ThreadProcSafe` arka plan iş parçacığı tarafından yürütülen yöntemi. Varsa <xref:System.Windows.Forms.TextBox> denetimin <xref:System.Windows.Forms.Control.InvokeRequired%2A> döndürür `true`, `ThreadProcSafe` yöntemi bir örneğini oluşturur `StringArgReturningVoidDelegate` ve formun geçen <xref:System.Windows.Forms.Control.Invoke%2A> yöntemi. Bu neden `SetText` oluşturulan iş parçacığı üzerinde çağrılacak yöntemi <xref:System.Windows.Forms.TextBox> denetimi ve bu iş parçacığı bağlamda <xref:System.Windows.Forms.Control.Text%2A> özelliği doğrudan ayarlayın.  
  
```csharp  
// This event handler creates a thread that calls a   
// Windows Forms control in a thread-safe way.  
private void setTextSafeBtn_Click(  
    object sender,   
    EventArgs e)  
{  
    this.demoThread =   
        new Thread(new ThreadStart(this.ThreadProcSafe));  
  
    this.demoThread.Start();  
}  
  
// This method is executed on the worker thread and makes  
// a thread-safe call on the TextBox control.  
private void ThreadProcSafe()  
{  
    this.SetText("This text was set safely.");  
}  
```


```cpp  
// This delegate enables asynchronous calls for setting  
// the text property on a TextBox control.  
delegate void StringArgReturningVoidDelegate(String^ text);  
```  
  
```csharp  
// This method demonstrates a pattern for making thread-safe  
// calls on a Windows Forms control.   
//  
// If the calling thread is different from the thread that  
// created the TextBox control, this method creates a  
// StringArgReturningVoidDelegate and calls itself asynchronously using the  
// Invoke method.  
//  
// If the calling thread is the same as the thread that created  
// the TextBox control, the Text property is set directly.   
  
private void SetText(string text)  
{  
    // InvokeRequired required compares the thread ID of the  
    // calling thread to the thread ID of the creating thread.  
    // If these threads are different, it returns true.  
    if (this.textBox1.InvokeRequired)  
    {     
        StringArgReturningVoidDelegate d = new StringArgReturningVoidDelegate(SetText);  
        this.Invoke(d, new object[] { text });  
    }  
    else  
    {  
        this.textBox1.Text = text;  
    }  
}  
```  
  
```vb  
' This method demonstrates a pattern for making thread-safe  
' calls on a Windows Forms control.   
'  
' If the calling thread is different from the thread that  
' created the TextBox control, this method creates a  
' StringArgReturningVoidDelegate and calls itself asynchronously using the  
' Invoke method.  
'  
' If the calling thread is the same as the thread that created  
 ' the TextBox control, the Text property is set directly.  
  
 Private Sub SetText(ByVal [text] As String)  
  
     ' InvokeRequired required compares the thread ID of the  
     ' calling thread to the thread ID of the creating thread.  
     ' If these threads are different, it returns true.  
     If Me.textBox1.InvokeRequired Then  
         Dim d As New StringArgReturningVoidDelegate(AddressOf SetText)  
         Me.Invoke(d, New Object() {[text]})  
     Else  
         Me.textBox1.Text = [text]  
     End If  
 End Sub  
```  
  
```cpp  
// This method demonstrates a pattern for making thread-safe  
    // calls on a Windows Forms control.  
    //  
    // If the calling thread is different from the thread that  
    // created the TextBox control, this method creates a  
    // StringArgReturningVoidDelegate and calls itself asynchronously using the  
    // Invoke method.  
    //  
    // If the calling thread is the same as the thread that created  
    // the TextBox control, the Text property is set directly.  
  
private:  
    void SetText(String^ text)

        // calling thread to the thread ID of the creating thread.

        }  
        else  
        {  
            this->textBox1->Text = text;  
        }  
    }  
```  
  
## <a name="making-thread-safe-calls-by-using-backgroundworker"></a>BackgroundWorker kullanarak iş parçacığı güvenli aramalar yapma  
 Tercih edilen yol için uygulama uygulamanızda çoklu iş parçacığı kullanımı kullanmaktır <xref:System.ComponentModel.BackgroundWorker> bileşeni. <xref:System.ComponentModel.BackgroundWorker> Bileşeni kullanan bir olay denetimli modeli için çoklu iş parçacığı kullanımı. Arka plan iş parçacığı çalıştırır, <xref:System.ComponentModel.BackgroundWorker.DoWork> olay işleyicisi ve denetimleri çalışmalarınız oluşturur iş parçacığı, <xref:System.ComponentModel.BackgroundWorker.ProgressChanged> ve <xref:System.ComponentModel.BackgroundWorker.RunWorkerCompleted> olay işleyicileri. Denetimlerden çağırabilir, <xref:System.ComponentModel.BackgroundWorker.ProgressChanged> ve <xref:System.ComponentModel.BackgroundWorker.RunWorkerCompleted> olay işleyicileri.  
  
#### <a name="to-make-thread-safe-calls-by-using-backgroundworker"></a>BackgroundWorker kullanarak iş parçacığı güvenli aramalar yapma  
  
1.  Arka plan iş parçacığında istediğiniz işlerini yapmak için bir yöntem oluşturma. Bu yöntem ana iş parçacığı tarafından oluşturulan denetimleri çağırmayın.  
  
2.  Arka plan iş sonuçlarını sona erdikten sonra rapor için bir yöntem oluşturun. Bu yöntem ana iş parçacığı tarafından oluşturulan denetimleri çağırabilirsiniz.  
  
3.  Adım 1'de oluşturulan yöntemi bağlamak <xref:System.ComponentModel.BackgroundWorker.DoWork> olay örneği <xref:System.ComponentModel.BackgroundWorker>ve bind aynı örneğinin için 2. adımda oluşturulan yöntemi <xref:System.ComponentModel.BackgroundWorker.RunWorkerCompleted> olay.  
  
4.  Arka plan iş parçacığı başlatmak için arama <xref:System.ComponentModel.BackgroundWorker.RunWorkerAsync%2A> yöntemi <xref:System.ComponentModel.BackgroundWorker> örneği.  
  
 Aşağıdaki kod örneğinde, <xref:System.ComponentModel.BackgroundWorker.DoWork> olay işleyicisi kullanan <xref:System.Threading.Thread.Sleep%2A> bazı Süren iş benzetimini yapmak için. Formun çağırmaz <xref:System.Windows.Forms.TextBox> denetim. <xref:System.Windows.Forms.TextBox> Denetimin <xref:System.Windows.Forms.Control.Text%2A> özelliği ayarlanmış doğrudan <xref:System.ComponentModel.BackgroundWorker.RunWorkerCompleted> olay işleyicisi.  
  
```csharp  
// This BackgroundWorker is used to demonstrate the   
// preferred way of performing asynchronous operations.  
private BackgroundWorker backgroundWorker1;  
```  
  
```vb

' preferred way of performing asynchronous operations.

```  
  
```cpp

    // preferred way of performing asynchronous operations.

```  
  
```csharp  
// This event handler starts the form's   
// BackgroundWorker by calling RunWorkerAsync.  
//  
// The Text property of the TextBox control is set  
// when the BackgroundWorker raises the RunWorkerCompleted  
// event.  
private void setTextBackgroundWorkerBtn_Click(  
    object sender,   
    EventArgs e)  
{  
    this.backgroundWorker1.RunWorkerAsync();  
}  
  
// This event handler sets the Text property of the TextBox  
// control. It is called on the thread that created the   
// TextBox control, so the call is thread-safe.  
//  
// BackgroundWorker is the preferred way to perform asynchronous  
// operations.  
  
private void backgroundWorker1_RunWorkerCompleted(  
    object sender,   
    RunWorkerCompletedEventArgs e)  
{  
    this.textBox1.Text =   
        "This text was set safely by BackgroundWorker.";  
}
```  

## <a name="example"></a>Örnek  
 Aşağıdaki kod örneğinde bir formu üç düğme ve bir metin kutusu içeren tam bir Windows Forms uygulamasıdır. Güvenli olmayan iş parçacıkları arası erişim ilk düğmesini gösterir, ikinci düğme kullanarak güvenli erişim gösterir <xref:System.Windows.Forms.Control.Invoke%2A>, üçüncü düğmesini kullanarak güvenli erişim gösterir <xref:System.ComponentModel.BackgroundWorker>.  
  
> [!NOTE]
>  Örneği çalıştırmak yönergeler için bkz: [nasıl yapılır: derleme ve çalıştırma bir tam Windows Forms kod örneği kullanarak Visual Studio](http://msdn.microsoft.com/en-us/cc447f7e-4c3b-4397-9d05-aeba3ca49416). Bu örnek, System.Drawing ve System.Windows.Forms derlemelerine başvurular gerektirir.

```csharp  
        }

        {  
            if (disposing && (components != null))  
            {

        }

            object sender,   
            EventArgs e)  
        {

                new Thread(new ThreadStart(this.ThreadProcUnsafe));  
  
            this.demoThread.Start();

        }

        // a thread-safe call on the TextBox control.  
        private void ThreadProcSafe()

        }

        // created the TextBox control, this method creates a  
        // StringArgReturningVoidDelegate and calls itself asynchronously using the

            if (this.textBox1.InvokeRequired)

                this.Invoke(d, new object[] { text });  
            }  
            else  
            {  
                this.textBox1.Text = text;  
            }  
        }  
  
        // This event handler starts the form's   
        // BackgroundWorker by calling RunWorkerAsync.  
        //  
        // The Text property of the TextBox control is set  
        // when the BackgroundWorker raises the RunWorkerCompleted  
        // event.  
        private void setTextBackgroundWorkerBtn_Click(  
            object sender,   
            EventArgs e)  
        {  
            this.backgroundWorker1.RunWorkerAsync();  
        }  
  
        // This event handler sets the Text property of the TextBox  
        // control. It is called on the thread that created the   
        // TextBox control, so the call is thread-safe.  
        //  
        // BackgroundWorker is the preferred way to perform asynchronous  
        // operations.  
  
        private void backgroundWorker1_RunWorkerCompleted(  
            object sender,   
            RunWorkerCompletedEventArgs e)  
        {

                "This text was set safely by BackgroundWorker.";  
        }
```  
