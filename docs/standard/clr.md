 <span data-ttu-id="3f2b0-121">Bunların meta verilerini bir parçası olarak, tüm yönetilen bileşenleri oluşturuldukları kaynakları ve bileşenleri hakkında bilgi taşır.</span><span class="sxs-lookup"><span data-stu-id="3f2b0-121">As part of their metadata, all managed components carry information about the components and resources they were built against.</span></span> <span data-ttu-id="3f2b0-122">Çalışma zamanı bileşeni veya uygulama, gereken her şeyi belirtilen sürümleri kodunuzu bazı unmet bağımlılık nedeniyle bölün olasılığı kılan olduğundan emin olmak için bu bilgileri kullanır.</span><span class="sxs-lookup"><span data-stu-id="3f2b0-122">The runtime uses this information to ensure that your component or application has the specified versions of everything it needs, which makes your code less likely to break because of some unmet dependency.</span></span> <span data-ttu-id="3f2b0-123">Kayıt bilgileri ve durumu verileri artık burada bunlar kurmak ve bakımını zor olabilir kayıt defterinde depolanır.</span><span class="sxs-lookup"><span data-stu-id="3f2b0-123">Registration information and state data are no longer stored in the registry where they can be difficult to establish and maintain.</span></span> <span data-ttu-id="3f2b0-124">Bunun yerine, tanımladığınız türleri (ve bağımlılıklarını) hakkında bilgi koduyla bileşen çoğaltma ve temizleme görevleri daha az karmaşık hale meta veri depolanır.</span><span class="sxs-lookup"><span data-stu-id="3f2b0-124">Instead, information about the types you define (and their dependencies) is stored with the code as metadata, making the tasks of component replication and removal much less complicated.</span></span>
