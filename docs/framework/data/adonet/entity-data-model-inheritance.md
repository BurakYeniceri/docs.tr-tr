# <a name="entity-data-model-inheritance"></a><span data-ttu-id="164e3-102">Varlık veri modeli: devralma</span><span class="sxs-lookup"><span data-stu-id="164e3-102">Entity Data Model: Inheritance</span></span>
<span data-ttu-id="164e3-103">Varlık veri modeli (EDM) için devralma destekleyen [varlık türleri](../../../../docs/framework/data/adonet/entity-type.md).</span><span class="sxs-lookup"><span data-stu-id="164e3-103">The Entity Data Model (EDM) supports inheritance for [entity types](../../../../docs/framework/data/adonet/entity-type.md).</span></span> <span data-ttu-id="164e3-104">EDM devralma nesne odaklı programlama dillerinde sınıfları için devralma benzer.</span><span class="sxs-lookup"><span data-stu-id="164e3-104">Inheritance in the EDM is similar to inheritance for classes in object-oriented programming languages.</span></span> <span data-ttu-id="164e3-105">Nesne odaklı dillerde sınıflarıyla kavramsal modelde bir varlık türü tanımlayabilirsiniz gibi (bir *türetilen türün*) başka bir varlık türünden devralan ( *temel türü*).</span><span class="sxs-lookup"><span data-stu-id="164e3-105">Like with classes in object-oriented languages, in a conceptual model you can define an entity type (a *derived type*) that inherits from another entity type (the *base type*).</span></span> <span data-ttu-id="164e3-106">Ancak, nesne odaklı programlama sınıflarında, kavramsal modelde türetilmiş bir tür her zaman tüm devralır [özellikleri](../../../../docs/framework/data/adonet/property.md) ve [Gezinti özellikleri](../../../../docs/framework/data/adonet/navigation-property.md) temel türü.</span><span class="sxs-lookup"><span data-stu-id="164e3-106">However, unlike classes in object-oriented programming, in a conceptual model the derived type always inherits all the [properties](../../../../docs/framework/data/adonet/property.md) and [navigation properties](../../../../docs/framework/data/adonet/navigation-property.md) of the base type.</span></span> <span data-ttu-id="164e3-107">Türetilmiş bir tür devralınan özelliklerinde geçersiz kılamaz.</span><span class="sxs-lookup"><span data-stu-id="164e3-107">You cannot override inherited properties in a derived type.</span></span>  
  
 <span data-ttu-id="164e3-108">Kavramsal modelde devralma Hiyerarşiler türetilmiş bir tür başka bir türetilmiş türden devralan oluşturabilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="164e3-108">In a conceptual model you can build inheritance hierarchies in which a derived type inherits from another derived type.</span></span> <span data-ttu-id="164e3-109">(Türetilmiş bir tür değil hiyerarşideki bir türü) hiyerarşinin üstündeki türü olarak adlandırılır *kök türü*.</span><span class="sxs-lookup"><span data-stu-id="164e3-109">The type at the top of the hierarchy (the one type in the hierarchy that is not a derived type) is called the *root type*.</span></span> <span data-ttu-id="164e3-110">Bir devralma hiyerarşisinde [Varlık anahtarı](../../../../docs/framework/data/adonet/entity-key.md) kök türünde tanımlanmış olması gerekir.</span><span class="sxs-lookup"><span data-stu-id="164e3-110">In an inheritance hierarchy, the [entity key](../../../../docs/framework/data/adonet/entity-key.md) must be defined on the root type.</span></span>  
  
 <span data-ttu-id="164e3-111">Devralma Hiyerarşiler türetilmiş bir tür birden fazla türünden devralan oluşturamaz.</span><span class="sxs-lookup"><span data-stu-id="164e3-111">You cannot build inheritance hierarchies in which a derived type inherits from more than one type.</span></span> <span data-ttu-id="164e3-112">Örneğin, bir modeldeki kavramsal ile bir `Book` varlık türü, türetilmiş türler tanımlayabilirsiniz `FictionBook` ve `NonFictionBook` her öğesinden devralmalı `Book`.</span><span class="sxs-lookup"><span data-stu-id="164e3-112">For example, in a conceptual model with a `Book` entity type, you could define derived types `FictionBook` and `NonFictionBook` that each inherit from `Book`.</span></span> <span data-ttu-id="164e3-113">Ancak, daha sonra hem de devralan bir tür tanımlayabilirsiniz değil `FictionBook` ve `NonFictionBook` türleri.</span><span class="sxs-lookup"><span data-stu-id="164e3-113">However, you could not then define a type that inherits from both the `FictionBook` and `NonFictionBook` types.</span></span>
