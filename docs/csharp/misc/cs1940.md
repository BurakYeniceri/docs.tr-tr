 <span data-ttu-id="cc0de-105">Bu hata, bir sorgu yöntemi, birden çok uygulamalarını tanımlanır ve derleyici hangisinin sorgu için kullanmak en iyisidir ayırt etmek olamaz oluşturulur.</span><span class="sxs-lookup"><span data-stu-id="cc0de-105">This error is generated when multiple implementations of a query method are defined and the compiler cannot disambiguate which one is best to use for the query.</span></span> <span data-ttu-id="cc0de-106">Aşağıdaki örnekte, her iki sürümü `Select` çünkü her ikisi de kabul aynı imzaya sahip `int` giriş parametresi olarak ve `int` dönüş değeri olarak.</span><span class="sxs-lookup"><span data-stu-id="cc0de-106">In the following example, both versions of `Select` have the same signature, because they both accept one `int` as an input parameter and have `int` as a return value.</span></span>

## <a name="example"></a><span data-ttu-id="cc0de-109">Örnek</span><span class="sxs-lookup"><span data-stu-id="cc0de-109">Example</span></span>  
 <span data-ttu-id="cc0de-110">Aşağıdaki kod CS1940 oluşturur:</span><span class="sxs-lookup"><span data-stu-id="cc0de-110">The following code generates CS1940:</span></span>  
  
```csharp  
// cs1940.cs  
using System; //must include explicitly for types defined in 3.5  
class Test  
{  
    public delegate int Dele(int x);  
    int num = 0;  
    public int Select(Func<int, int> d)  
    {  
        return d(this.num);  
    }  
    public int Select(Dele d) // CS1940  
    {  
        return d(this.num) + 1;  
    }  
    public static void Main()  
    {  
        var q = from x in new Test()  
        select x;  
    }  
}  
```
