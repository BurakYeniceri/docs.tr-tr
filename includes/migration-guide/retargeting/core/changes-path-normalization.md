### <a name="changes-in-path-normalization"></a>Yol normalleştirme değişiklikleri

|   |   |
|---|---|
|Ayrıntılar|.NET Framework 4.6.2 hedefleyen uygulamalar ile başlayarak, çalışma zamanı yolları normalleştirir şekilde değiştirdi. Bir yol normalleştirme bir yol veya dosya tanımlar ve böylece geçerli bir yol hedef işletim sisteminde uyacağını dizesini değiştirmeyi içerir. Normalleştirme genellikle içerir:<ul><li>Bileşen ve dizin ayırıcı standart hale getirme.</li><li>Geçerli dizine göreli bir yol uygulanıyor.</li><li>Göreli dizin (.) veya bir yol üst dizinindeki (.) değerlendiriliyor.</li><li>Kırpma karakterleri belirtilmiş.</li></ul>.NET Framework 4.6.2 hedefleyen uygulamalar ile başlayarak, yol normalleştirme aşağıdaki değişiklikleri varsayılan olarak etkindir:<ul><li>İşletim sistemi için çalışma zamanı erteler [GetFullPathName](https://msdn.microsoft.com/library/windows/desktop/aa364963(v=vs.85).aspx) yolları normalleştirmek için işlevi.</li><li>Normalleştirme artık dizin Segment (örneğin, bir dizin adının sonunda boşluk) sonuna kırpma içerir.</li><li>Tam güven yolu sözdiziminde cihaz için destek dahil olmak üzere <code>\\.\</code> and, for file I/O APIs in mscorlib.dll, '\?'.</li><li>The runtime does not validate device syntax paths.</li><li>The use of device syntax to access alternate data streams is supported.</li></ul>These changes improve performance while allowing methods to access previously inaccessible paths. Apps that target the .NET Framework 4.6.1 and earlier versions but are running under the .NET Framework 4.6.2 or later are unaffected by this change.|
|Öneri|.NET Framework 4.6.2 hedefleyen veya daha sonra bu dışında seçebilirsiniz uygulamalar değiştirin ve aşağıdaki ekleyerek eski normalleştirme kullanın <code>&lt;runtime&gt;</code> uygulama yapılandırma dosyasının:<pre><code class="language-xml">&lt;runtime&gt;&#13;&#10;&lt;AppContextSwitchOverrides value=&quot;Switch.System.IO.UseLegacyPathHandling=true&quot; /&gt;&#13;&#10;&lt;/runtime&gt;&#13;&#10;</code></pre>.NET Framework 4.6.1 veya önceki ancak hedef uygulamaları aşağıdaki satırı ekleyerek yolu normalleştirme değişiklikleri daha sonra etkinleştirebilirsiniz veya .NET Framework 4.6.2 çalıştırıyorsanız <code>&lt;runtime&gt;</code> uygulama .configuration dosyasının:<pre><code class="language-xml">&lt;runtime&gt;&#13;&#10;&lt;AppContextSwitchOverrides value=&quot;Switch.System.IO.UseLegacyPathHandling=false&quot; /&gt;&#13;&#10;&lt;/runtime&gt;&#13;&#10;</code></pre>|
|Kapsam|Küçük|
|Sürüm|4.6.2|
|Tür|Yeniden hedefleme|
