### <a name="what-does-this-mean-for-client-scenario"></a><span data-ttu-id="28047-171">Bu ne istemci senaryo için anlama geliyor?</span><span class="sxs-lookup"><span data-stu-id="28047-171">What does this mean for client scenario?</span></span>

<span data-ttu-id="28047-176">Ayrıca, (örneğin, bir kullanıcı Arabirimi güncelleştirme) kullanıcı Arabirimi iş parçacığı için iş gönderme ile çok basit olup `async` yöntemleri ve (örneğin, bir iş parçacığı temsilci çağırma) ek iş gerektirmez.</span><span class="sxs-lookup"><span data-stu-id="28047-176">Additionally, dispatching work to the UI thread (such as updating a UI) is very simple with `async` methods, and does not require extra work (such as calling a thread-safe delegate).</span></span>

## <a name="deeper-dive-into-task-and-taskt-for-a-cpu-bound-operation"></a><span data-ttu-id="28047-177">Derin Dalış görev ve görev<T> CPU bağımlı işlemi</span><span class="sxs-lookup"><span data-stu-id="28047-177">Deeper Dive into Task and Task<T> for a CPU-Bound Operation</span></span>

<span data-ttu-id="28047-178">CPU bağımlı `async` kodu g/Ç-sınırdan biraz farklıdır `async` kodu.</span><span class="sxs-lookup"><span data-stu-id="28047-178">CPU-bound `async` code is a bit different than I/O-bound `async` code.</span></span>  <span data-ttu-id="28047-179">İş CPU'da yapıldığından, hesaplama için bir iş parçacığı ayrılması geçici almak için yolu yoktur.</span><span class="sxs-lookup"><span data-stu-id="28047-179">Because the work is done on the CPU, there's no way to get around dedicating a thread to the computation.</span></span>  <span data-ttu-id="28047-180">Kullanımını `async` ve `await` bir arka plan ile etkileşim kurmak için temiz bir şekilde sizinle iş parçacığı ve zaman uyumsuz yöntem arayan yanıt verebilir durumda tutun sağlar.</span><span class="sxs-lookup"><span data-stu-id="28047-180">The use of `async` and `await` provides you with a clean way to interact with a background thread and keep the caller of the async method responsive.</span></span>  <span data-ttu-id="28047-181">Bu paylaşılan veriler için herhangi bir koruma sağlamaz unutmayın.</span><span class="sxs-lookup"><span data-stu-id="28047-181">Note that this does not provide any protection for shared data.</span></span>  <span data-ttu-id="28047-182">Paylaşılan veri kullanıyorsanız, uygun eşitleme stratejisi uygulamak gerekecektir.</span><span class="sxs-lookup"><span data-stu-id="28047-182">If you are using shared data, you will still need to apply an appropriate synchronization strategy.</span></span>

<span data-ttu-id="28047-183">CPU bağımlı zaman uyumsuz çağrı 10.000 kaplama alanı görünümünü şöyledir:</span><span class="sxs-lookup"><span data-stu-id="28047-183">Here's a 10,000 foot view of a CPU-bound async call:</span></span>

```csharp
public async Task<int> CalculateResult(InputData data)
{
    // This queues up the work on the threadpool.
    var expensiveResultTask = Task.Run(() => DoExpensiveCalculation(data));
    
    // Note that at this point, you can do some other work concurrently,
    // as CalculateResult() is still executing!
    
    // Execution of CalculateResult is yielded here!
    var result = await expensiveResultTask;
    
    return result;
}
```

<span data-ttu-id="28047-184">`CalculateResult()`çağrıldı iş parçacığı üzerinde yürütür.</span><span class="sxs-lookup"><span data-stu-id="28047-184">`CalculateResult()` executes on the thread it was called on.</span></span>  <span data-ttu-id="28047-185">Bunu çağırdığında `Task.Run`, pahalı CPU bağımlı işlemi kuyruklar `DoExpensiveCalculation()`, iş parçacığı havuzu üzerinde ve alan bir `Task<int>` işlemek.</span><span class="sxs-lookup"><span data-stu-id="28047-185">When it calls `Task.Run`, it queues the expensive CPU-bound operation, `DoExpensiveCalculation()`, on the thread pool and receives a `Task<int>` handle.</span></span>  <span data-ttu-id="28047-186">`DoExpensiveCalculation()`sonunda eşzamanlı olarak sonraki kullanılabilir iş parçacığı üzerinde başka bir CPU çekirdeği üzerinde büyük olasılıkla çalıştırılır.</span><span class="sxs-lookup"><span data-stu-id="28047-186">`DoExpensiveCalculation()` is eventually run concurrently on the next available thread, likely on another CPU core.</span></span>  <span data-ttu-id="28047-187">Eşzamanlı iş çalışırken yapmak mümkündür `DoExpensiveCalculation()` başka bir iş parçacığı üzerinde meşgul olduğundan çağrılan iş parçacığı `CalculateResult()` yürütülmeye devam.</span><span class="sxs-lookup"><span data-stu-id="28047-187">It's possible to do concurrent work while `DoExpensiveCalculation()` is busy on another thread, because the thread which called `CalculateResult()` is still executing.</span></span>

<span data-ttu-id="28047-188">Bir kez `await` karşılaşılırsa, yürütülmesi `CalculateResult()` sırasında geçerli iş parçacığı ile yapılacak diğer işleri izin vererek, arayana verdiğini `DoExpensiveCalculation()` bir sonuç fakat.</span><span class="sxs-lookup"><span data-stu-id="28047-188">Once `await` is encountered, the execution of `CalculateResult()` is yielded to its caller, allowing other work to be done with the current thread while `DoExpensiveCalculation()` is churning out a result.</span></span>  <span data-ttu-id="28047-189">Tamamlandıktan sonra sonuç ana iş parçacığında çalıştırmak için sıraya alındı.</span><span class="sxs-lookup"><span data-stu-id="28047-189">Once it has finished, the result is queued up to run on the main thread.</span></span>  <span data-ttu-id="28047-190">Sonuç olarak, ana iş parçacığı yürütülen döndürülecek `CalculateResult()`, bu noktada sonucu olacaktır `DoExpensiveCalculation()`.</span><span class="sxs-lookup"><span data-stu-id="28047-190">Eventually, the main thread will return to executing `CalculateResult()`, at which point it will have the result of `DoExpensiveCalculation()`.</span></span>
